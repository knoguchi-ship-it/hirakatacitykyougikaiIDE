# リリース記録 v240（2026-04-19）

## 概要
v240 では、管理者 Google セッションで会員マイページへ遷移した際に `認証アカウントが見つかりません` と表示される不具合を修正した。原因は、管理者セッションの Google メールを会員マイページ用 `loginId` として流用していたデータモデル不整合であり、今回の修正で識別子を分離した。

あわせて、管理者 Google ホワイトリスト更新時に `admin_auth_v1` キャッシュが残留していたため、権限・アカウント紐付け変更が即時反映されない経路も修正した。以後、管理権限変更は同一営業日待ちではなく即時反映を前提に扱う。

## 変更内容

### 1. 管理者セッション識別子の分離
- フロントエンドの認証コンテキストを `memberPortalLoginId` と `memberId/staffId` ベースに整理。
- ID/パスワード会員だけが `memberPortalLoginId` を保持し、管理者 Google セッションでは保持しない。
- 管理者ログイン後に会員マイページを再読込する場合、`getMemberPortalData` は `memberId` で呼び出すよう変更。

### 2. 会員ポータル取得 API の入力モデル修正
- `getMemberPortalData` を `loginId` 固定から `loginId | memberId` の lookup モデルへ変更。
- backend `getMemberPortalData_()` の既存 `memberId` 後方互換経路を、管理者セッションの正規経路として利用する形に整理。

### 3. 管理権限変更の即時反映
- `clearAdminPermissionCaches_()` を追加。
- `T_管理者Googleホワイトリスト` の保存・削除・リンク修復時に、`admin_wl_v1` と `admin_auth_v1` の両方を無効化。
- これにより、Google ホワイトリストと認証アカウント紐付けの変更が最大 300 秒遅延する問題を解消。

### 4. 正本ドキュメント更新
- `docs/05_AUTH_AND_ROLE_SPEC.md` に、`PASSWORD` ログインIDと管理者 Google メールを同一 `loginId` として混用しない原則を追記。
- `docs/03_DATA_MODEL.md` に、管理者セッションから会員マイページを再読込する際の解決キーを `memberId/staffId` とする運用を追記。
