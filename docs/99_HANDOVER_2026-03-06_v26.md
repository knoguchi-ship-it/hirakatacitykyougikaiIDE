## 6. DB 移行手順（本番スプレッドシートへの新列追加）

T_研修 に 9 列追加されました。本番 DB には **まだ反映されていません**。
以下のいずれかの手順で対応してください。

### 方法A: スキーマ再構築（データ保持）
```bash
npx clasp run rebuildDatabaseSchema
```
※ ヘッダー行の末尾に新列が追加されます。既存データは保持されます。

### 方法B: シードデータ再投入（データリセット）
```bash
npx clasp run seedDemoData
```
※ 既存のすべてのテーブルデータがクリアされ、デモデータが再投入されます。
※ デモ環境のみ使用可。本番データがある場合は方法 A を使用してください。

## 7. 重要な運用制約

| 制約 | 詳細 |
|---|---|
| Web App デプロイ | `clasp deploy` は **API Executable** になる。Web App は **Apps Script UI** からのみ更新可能 |
| clasp push | `npx clasp push --force` が必要（`--force` なしは skip される場合あり） |
| ファイルアップロード | GAS の `google.script.run` は戻り値制限あり。ファイルは base64 で渡し Drive 保存 |
| アップロード上限 | 5MB 推奨（GAS の `Utilities.base64Decode` のメモリ制限） |
| Drive フォルダ | 「研修案内状」フォルダ（自動作成）。共有設定: リンクを知っている全員が閲覧可 |
