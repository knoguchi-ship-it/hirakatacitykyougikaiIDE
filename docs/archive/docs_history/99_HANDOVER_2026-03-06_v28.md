## 3. 本セッション（v27→v28）での実装内容

### 3-1. 動的費用区分（複数費用登録）

研修の費用を「会員・非会員・学生」等の複数区分で動的に登録できるようにしました。

**変更前**: 単一の `fee: number` フィールド（DB列: `費用`）
**変更後**: `fees: TrainingFee[]` 配列（DB列: `費用JSON` に JSON 文字列として保存）

```typescript
export interface TrainingFee {
  label: string;  // 例: "会員", "非会員"
  amount: number; // 円
}
```

UI: ラベル入力 + 金額入力の行を動的追加・削除可能。初期値は「会員 ¥0」「非会員 ¥0」。

### 3-2. フォーム項目 ON/OFF 設定パネル

研修登録フォームの上部に折りたたみ式「⚙ 表示する項目を設定」パネルを追加。
オプション 10 項目をトグルスイッチで ON/OFF できます。

**トグル可能な項目**:
| key | ラベル |
|---|---|
| `organizer` | 主催者 |
| `summary` | 研修概要 |
| `description` | 研修内容 |
| `location` | 開催場所 |
| `instructor` | 講師 |
| `applicationOpenDate` | 参加受付開始日 |
| `applicationCloseDate` | 参加締め切り日 |
| `fees` | 費用区分 |
| `isNonMandatory` | 法定外研修フラグ |
| `guidePdfUrl` | 案内状ファイル |

OFF にした項目はフォームから非表示になり、保存時も送信されません。
設定は `fieldConfig` として DB の `項目設定JSON` 列に JSON 保存されます。

### 3-3. 型定義追加（`src/types.ts`）

```typescript
export interface TrainingFieldConfig {
  organizer: boolean;
  isNonMandatory: boolean;
  summary: boolean;
  description: boolean;
  location: boolean;
  instructor: boolean;
  applicationOpenDate: boolean;
  applicationCloseDate: boolean;
  fees: boolean;
  guidePdfUrl: boolean;
}

export const DEFAULT_FIELD_CONFIG: TrainingFieldConfig = {
  // 全項目 true（全 ON）
};

export const DEFAULT_FEES: TrainingFee[] = [
  { label: '会員', amount: 0 },
  { label: '非会員', amount: 0 },
];

// Training 型に追加
fees?: TrainingFee[];
fieldConfig?: TrainingFieldConfig;
```

### 3-5. GAS ロジック変更

`fetchAllDataFromDb_` の研修マッピング:
- `費用JSON` を `JSON.parse()` で配列に変換
- パース失敗時は旧数値データとして後方互換フォールバック
- `項目設定JSON` を `JSON.parse()` で `fieldConfig` に変換

`saveTraining_` の保存:
- `fees` を `JSON.stringify()` して `費用JSON` 列へ
- `fieldConfig` を `JSON.stringify()` して `項目設定JSON` 列へ

### 3-6. ビルド手順の確認事項（重要）

GAS へのデプロイには必ず **`npm run build:gas`** を使用してください。
`npm run build`（`vite build` のみ）では `backend/index.html` が更新されません。

```bash
# 正しい手順
npm run build:gas   # vite build + dist/index.html → backend/index.html コピー
npx clasp push --force
# その後 Apps Script UI でデプロイ更新
```
