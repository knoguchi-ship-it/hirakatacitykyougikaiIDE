# Release State: v261

リリース日: 2026-04-23
GAS versions: 統合 262 / 会員 split 14 / 管理者 split 20

## 変更内容

### 1. ログテーブル別スプレッドシート分離

- `getLogSs_()` を追加: `LOG_SPREADSHEET_ID` スクリプトプロパティが設定されていればログ SS を使用。未設定時はメイン DB にフォールバックするため、移行前後で動作は変わらない（安全移行）。
- 対象テーブル: `T_ログイン履歴` / `T_監査ログ` / `T_メール送信ログ`
- 移行用関数: `setupLogSpreadsheet()` / `rebuildLogDatabaseSchema()` / `migrateLogsToLogSpreadsheet()`

### 2. 退会済み会員の定期 archive 移動

- `T_会員_archive` / `T_事業所職員_archive` シートを追加（テーブル定義に追加）
- `runArchiveOldWithdrawnMembers()`: 退会日から 3 年以上経過した WITHDRAWN 会員を archive シートに移動
- `moveWithdrawnRowsToArchive_()`: archive 移動ヘルパー関数

### 3. T_メール送信ログ書き込みバグ修正（本番バグ修正）

- `sendBulkMemberMail_()` 内の `appendRowsByHeaders_()` 呼び出しが引数不足（2引数）だったため、本番環境でログが一切書き込まれていなかった。3引数に修正。

### 4. 入会申込フォーム UI 改善（v258 同期）

- 事業所会員 step 3（職員入力）から削除ボタンを除去（10枠固定表示のため不要）
- 住所に「建物名・部屋番号」任意フィールドを追加（勤務先・自宅の両方）
- `types.ts` に `officeAddressLine2` / `homeAddressLine2` フィールドを追加

### 5. 管理者画面 職員追加カード UI 改善

- 1行9カラムの詰まったレイアウトを2行構成に変更（姓名カナメール / CM番号・役割・取消ボタン）
- 在籍/除籍 select を除去（追加時は常に在籍）
- 「追加を取消」ボタンを視覚的に目立つ赤枠スタイルに変更

## v261 移行手順

**注意**: LOG_SPREADSHEET_ID が設定されるまではフォールバック動作（メイン DB に記録）が継続する。移行は任意のタイミングで実施可能。

### ステップ 1: メイン DB にアーカイブシートを作成

```
npx clasp run rebuildDatabaseSchema
```
`T_会員_archive` と `T_事業所職員_archive` シートが作成される。

### ステップ 2: ログ SS を新規作成（1回のみ）

```
npx clasp run setupLogSpreadsheet
```
新しいスプレッドシートが作成され、ID がスクリプトプロパティ `LOG_SPREADSHEET_ID` に保存される。
ログに出力されるスプレッドシート ID を控えておく。

### ステップ 3: 他 2 プロジェクトにも同じ LOG_SPREADSHEET_ID を設定

Apps Script エディタ → プロジェクトの設定 → スクリプト プロパティ で
`LOG_SPREADSHEET_ID` = ステップ 2 で作成した SS ID
を会員 split と admin split にも設定する。

### ステップ 4: ログ SS にシートを作成

```
npx clasp run rebuildLogDatabaseSchema
```
ログ SS に `T_ログイン履歴` / `T_監査ログ` / `T_メール送信ログ` が作成される。

### ステップ 5: 既存ログデータを移行

```
npx clasp run migrateLogsToLogSpreadsheet
```
メイン DB の既存ログ行をログ SS にコピーする（元データは残す）。

### ステップ 6: archive 定期実行の設定（任意）

Apps Script エディタのトリガー画面で `runArchiveOldWithdrawnMembers` を
月次または週次の時間ベーストリガーとして登録する。
