# リリース記録 v243（2026-04-19）

## 概要

v243 では、事業所会員詳細の「職員を追加」行について、空白行で保存が詰まる問題と、必須入力が未反映に見える問題を修正した。

加えて、追加行のレイアウトを見直し、`取消` ボタンがモーダル幅によって画面外へ押し出されにくいようにした。

## 背景

- 事業所職員追加で誤って空白行を増やすと、保存時に既存の必須チェックへ先に入り、空白行を無視できなかった。
- メールアドレス必須や形式チェックは保存時にしか見えず、未実装に見えやすかった。
- 追加行が横一列前提のレイアウトで、`取消` ボタンが見切れるケースがあった。
- バックエンド `syncBusinessStaffRows_` は `氏名 / フリガナ` を前提にしており、フロントの新規職員行も同じ整合条件に合わせる必要があった。

## 実施内容

### 1. 新規職員ドラフト行の正規化順序を修正

- 保存処理の入口で `normalizeDraftStaffList()` を通すよう変更。
- `isNew && 完全空白` の行は、この時点で保存対象から除外。
- その後に必須チェックへ進むよう順序を整理。

### 2. 新規職員行の必須/形式チェックを明示化

- 新規職員行に対して以下を検証:
  - `氏名` 必須
  - `フリガナ` 必須
  - `メールアドレス` 必須
  - `メールアドレス` 形式
  - `介護支援専門員番号` 必須
  - `介護支援専門員番号` 8桁半角数字
- 行ごとのエラー状態 `draftStaffErrors` を追加。
- エラー時は入力欄を赤枠表示し、行内エラー要約も表示。

### 3. 新規職員行のレイアウト調整

- 追加行グリッドを `md:grid-cols-2` ベースへ変更。
- メール欄は横幅を維持しつつ、狭い幅では折り返しやすい構成へ変更。
- `取消` ボタンは `shrink-0` とし、押し出されにくいように調整。

### 4. ブラウザ標準 `required` を撤去

- 新規職員行の `required` 属性を外し、検証責務を React 側へ統一。
- これにより、空白行の無視ルールと独自バリデーションの整合性を確保した。
