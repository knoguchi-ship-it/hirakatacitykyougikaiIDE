# Release State: v216

Date: 2026-04-16
Version: `216`
Fixed deployments: member `@216` / public `@216`

---

## Summary

This release updates validation behavior for personal-member editing in both the member portal and the admin console.

- Validation errors are now easier to identify, with red field highlights, inline error messages, and a top-level error summary that jumps to the affected field.
- `セイ` / `メイ` now accept only half-width kana.
- `介護支援専門員番号` now accepts only 8 half-width digits, and remains required for individual members.
- For individual/support members, either `勤務先電話番号` or `携帯電話番号` is now required.
- When mail destination is `OFFICE`, only `事業所名` is required.
- When mail destination is `HOME`, only home `郵便番号 / 都道府県 / 市区町村 / 番地` are required.
- `FAX番号` is now optional.

## Changes

**Frontend (`src/components/MemberForm.tsx`)**
- Added input normalization for kana and care-manager number fields.
- Added new validation rules and error summary navigation for personal-member editing.
- Updated labels and field requirements to match the revised business rules.

**Frontend (`src/components/MemberDetailAdmin.tsx`)**
- Added equivalent validation rules and input normalization in the admin member-detail editor.
- Added red error styling, field-level messages, and a clickable error summary.
- Added scroll/focus behavior to the first invalid field on save.

**Backend (`backend/Code.gs`)**
- Updated `validateMemberPayload_()` to enforce the same rules server-side.
- Changed non-business mail-destination validation so `OFFICE` requires `officeName`, while `HOME` requires home address core fields.
- Removed legacy non-business FAX requirement.

**Docs**
- Updated `docs/05_AUTH_AND_ROLE_SPEC.md`
- Updated `docs/03_DATA_MODEL.md`

**Generated frontend assets**
