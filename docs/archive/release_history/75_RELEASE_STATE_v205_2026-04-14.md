## Summary

PDF名簿出力を1000件対応アーキテクチャへ完全刷新。
フロントエンドがチャンク分割し、GASは1回あたり最大250件・6分以内で処理。
all-or-nothing + リトライにより、全件成功のみ最終ZIPを出力する。

## Architecture

```
Frontend                           GAS Backend
  |                                    |
  | initRosterExport({year})           |
  |----------------------------------→| Drive 一時フォルダ作成
  |←-- { folderId }                   |
  |                                    |
  | processRosterChunk({               |
  |   folderId, chunkIndex:0,          |
  |   memberIds[0..249], year })       |
  |----------------------------------→| 250件処理 (PARALLEL_BATCH=15)
  |                                    | 失敗→MAX_RETRY=2回リトライ
  |                                    | 全成功: chunk_0.zip → Drive folder
  |←-- { ok:true, count:250 }         | 失敗残存: { ok:false, errors }
  |                                    |
  | processRosterChunk({chunkIndex:1}) |
  |----------------------------------→| 次の250件...
  |←-- { ok:true, count:250 }         |
  |                                    |
  | [チャンク失敗時]                    |
  | cleanupRosterExport({folderId})    |
  |----------------------------------→| 一時フォルダを削除
  |                                    |
  | finalizeRosterExport({folderId})  |
  |----------------------------------→| chunk_*.zip を unzip → rezip
  |                                    | 最終ZIP → Drive root保存
  |                                    | 一時フォルダ削除
  |←-- { downloadUrl, count, ... }    |
```

## Changes

### GAS backend (`backend/Code.gs`)

旧 `generateRosterZip_` を廃止し5関数に置き換え:

| 関数 | 役割 |
|------|------|
| `initRosterExport_` | Drive 一時フォルダ作成、folderId 返却 |
| `processRosterChunk_` | 250件/チャンク処理、MAX_RETRY=2、部分ZIP保存 |
| `finalizeRosterExport_` | 部分ZIP統合、最終ZIP生成、フォルダ削除 |
| `cleanupRosterExport_` | エラー・中断時のフォルダ削除 |
| `generatePdfsForIds_` | コアPDF生成ヘルパー、{blobs, failedIds, errors}を返す |

リトライ設計:
- `failedIds`: HTTP エラーのみ（transient）→ リトライ対象
- 会員データなし: errors のみ（永続エラー）→ リトライしない
- 3回失敗残存 → `{ok: false, errors}` を返し、フロントが cleanup して中断

### Frontend (`src/components/RosterExport.tsx`)

- `CHUNK_SIZE = 250` 定数追加
- `handleGenerate` をチャンクフローに刷新（init → chunk loop → finalize）
- エラー時の `cleanupRosterExport` 呼び出し
- `progress` state 追加（phase / current / total / processed）
- プログレスバー UI（チャンク進捗 + 件数カウント + 統合フェーズ表示）
- 部分成功エラー一覧 UI を削除（all-or-nothing のため不要）

### API client (`src/services/api.ts`)

- `generateRosterZip` を廃止
- `initRosterExport`, `processRosterChunk`, `finalizeRosterExport`, `cleanupRosterExport` を追加
