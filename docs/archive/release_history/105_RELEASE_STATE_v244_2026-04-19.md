# リリース記録 v244（2026-04-19）

## 概要

会員マイページ（MemberForm.tsx）の事業所職員追加 UX を修正。
v243 で管理者コンソール（MemberDetailAdmin.tsx）に適用した isNew ドラフト概念を、
会員マイページ側にも同様に実装した。

## 背景

スクリーンショットにて、会員マイページから事業所会員情報を変更しようとすると：
- 空白の「職員を追加」行が必須入力を要求してブロックする
- 追加した空白行を取り消せない（取消ボタン非表示）
- メールアドレスが必須表示されていない

v243 の修正は `MemberDetailAdmin.tsx`（管理者コンソール）にのみ適用されており、
`MemberForm.tsx`（会員マイページ）には未適用だった。
また、v239〜v243 の変更がすべて git 未コミットのままだったため、本コミットで一括確定した。

## 実施内容

### MemberForm.tsx

1. `type DraftStaff = Staff & { isNew?: boolean }` を追加
2. `EMAIL_RE` 正規表現を追加
3. `isBlankDraftStaff()` ヘルパーを追加
4. `addStaff()` で `isNew: true` をセット
5. `removeStaff()` を `removeNewStaff()` に変更（isNew 行専用・confirm 不要）
6. `validate()` で isNew かつ完全空白の行はスキップ、
   isNew かつ非空白の行には名前・フリガナ・メール必須チェックを適用
7. `handleSubmit()` で保存前に完全空白 isNew 行をフィルタアウト、`isNew` フラグを除去して保存
8. UI: isNew 行のメール欄に「※」を表示し、エラー時に赤枠・エラーメッセージを表示
9. UI: isNew 行に「取消」ボタンを追加（権限欄の代わりに表示）
10. UI: isNew 行は登録日・研修申込リンクを非表示
