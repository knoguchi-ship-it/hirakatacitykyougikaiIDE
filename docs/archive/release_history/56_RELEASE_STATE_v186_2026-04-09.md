# Release State: v186
リリース日: 2026-04-09
ブランチ: main
固定 deployment: member `@186` / public `@186`

## 変更概要
研修管理コンソール（`TrainingManagement.tsx`）の UX 改善とバックエンド対応。

## 変更内容

### フロントエンド

#### src/components/TrainingManagement.tsx
1. **費用非表示 = 参加費無料ラベル**
   - `fees` フィールドが fieldConfig で非表示の場合、フォーム内に緑色「参加費無料」バッジを表示。
   - 表示中の場合は従来どおり費用入力 UI を表示。

2. **申込日デフォルト自動入力**
   - `buildEmptyForm()` で申込開始日=当日・申込締切日=1か月後をデフォルト設定。
   - `applicationOpenDate` / `applicationCloseDate` が fieldConfig で有効な場合のみ適用。

3. **問い合わせ連絡先を電話番号・メールアドレスに分割**
   - `inquiryPhone` + `inquiryEmail` の 2 フィールド（3カラムグリッド：担当者・電話番号・メールアドレス）。
   - どちらか一方必須のバリデーション。
   - 旧レコード（`inquiryContactValue` のみ）は `inquiryContactType` をもとに自動マッピング。
   - 保存時は `inquiryPhone || inquiryEmail` を `inquiryContactValue` として維持（後方互換）。

4. **法定外研修フラグ常時表示**
   - fieldConfig の表示/非表示トグルから `isNonMandatory` を除外。
   - チェックボックスをフォーム内に常時表示。

5. **`TRAINING_OPTIONAL_FIELD_DEFS` export**
   - システム設定 UI で再利用するためエクスポート化。
   - 含む項目: 詳細説明・講師・申込開始日・申込締切日・研修費用・案内PDF（`isNonMandatory` は除外）。

#### src/App.tsx
- `trainingDefaultFieldConfig` / `trainingDefaultFieldConfigInput` state を追加。
- `applySystemSettings` で `trainingDefaultFieldConfig` を反映。
- `updateSystemSettings` call に `trainingDefaultFieldConfig` を含める。
- `TrainingManagement` に `defaultFieldConfig={trainingDefaultFieldConfig}` prop を渡す。
- システム設定画面に **研修フォーム 項目表示デフォルト設定** チェックボックスグリッドを追加。

#### src/types.ts
- `Training` に `inquiryPhone?: string` / `inquiryEmail?: string` を追加。
- `SystemSettings` に `trainingDefaultFieldConfig?: TrainingFieldConfig | null` を追加。

### バックエンド（backend/Code.gs）

- `parseTrainingOptions_`: `inquiryPhone` / `inquiryEmail` フィールドの読み込みと後方互換マッピングを追加。
- `serializeTrainingOptions_`: `inquiryPhone` / `inquiryEmail` の保存を追加。
- `normalizeInquiryContacts_(phone, email, legacyValue)` 新設: 新旧両形式を正規化し primary contact + type を返す。
- `saveTraining_`: `normalizeInquiryContacts_` を使用するよう変更。
- Training 読み込みパス: `inquiryPhone` / `inquiryEmail` を返却オブジェクトに追加。
- `getSystemSettings_`: `TRAINING_DEFAULT_FIELD_CONFIG` を JSON パースして `trainingDefaultFieldConfig` として返す。
- `updateSystemSettings_`: `trainingDefaultFieldConfig` を `TRAINING_DEFAULT_FIELD_CONFIG` キーとして保存。

## DB スキーマ変更
なし（`T_システム設定` の既存キー値テーブルに `TRAINING_DEFAULT_FIELD_CONFIG` を追加するのみ）。

## 後方互換
- 旧研修レコードは `inquiryContactValue` + `inquiryContactType` のみを持つ。フロントエンドのロード時に `inquiryPhone` / `inquiryEmail` へ自動マッピング。
- 新レコード保存時も `inquiryContactValue`（primaryContact）を維持するため、GAS 側でどちらの形式で読んでも破綻しない。
