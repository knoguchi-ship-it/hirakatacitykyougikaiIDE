# リリース記録 v239（2026-04-19）

## 概要
v239 では、事業所会員の職員追加・転籍でメールアドレスを必須化し、管理者/代表者が保存で詰まりやすかった新規職員行の扱いを改善した。将来の会員マイページ通知機能の前提として、事業所職員のメールアドレスを運用上の必須データに引き上げた。

## 変更内容

### 1. 事業所職員追加の保存バリデーション強化
- 新規職員行で `氏名` / `メールアドレス` / `介護支援専門員番号` を必須化。
- メールアドレス形式チェックを追加。
- 完全空白の新規追加行は保存対象から自動除外。
- 部分入力の行だけをエラー対象にすることで、不要に追加した空行が保存をブロックしないよう修正。

### 2. メール欄の表示幅改善
- 新規職員行のグリッドでメール列を `minmax(20rem, 1.8fr)` に拡張。
- 各入力に `w-full min-w-0` を適用し、メールアドレス欄が他列に押し潰されにくいよう調整。

### 3. 個人/賛助会員から事業所職員への転籍条件強化
- `convertIndividualToStaff_` で転籍元会員の `代表メールアドレス` を必須化。
- メール未登録の個人会員・賛助会員は、先にメール登録を行わない限り事業所職員へ転籍できない。
