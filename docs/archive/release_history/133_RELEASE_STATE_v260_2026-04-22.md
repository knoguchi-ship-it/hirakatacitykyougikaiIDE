## 3. 実装詳細

### 3.1 セキュリティ設計

| 項目 | 仕様 |
|---|---|
| 認証方式 | CM番号 → 登録メールへ 6 桁 OTP 送信 → 検証 → 単一使用アクショントークン |
| Enumeration 防止 | CM番号の有無を応答で露出しない（member/non-member で同一レスポンス） |
| OTP TTL | 600 秒（10 分）|
| OTP 試行上限 | 5 回超でキャッシュ削除（ロック） |
| 送信レート制限 | 15 分窓で 5 回まで（CacheService） |
| アクショントークン TTL | 1800 秒（30 分）|
| アクショントークン再利用 | 使用後即時削除（単一使用）|
| フィールド制限 | `PUBLIC_MEMBER_UPDATE_ALLOWLIST_` サーバーサイド allowlist |

### 3.2 新規公開 API（PUBLIC_ALLOWED_ACTIONS に追加）

| API | 用途 |
|---|---|
| `sendPublicOtp` | CM番号でメンバー検索 → 登録メールに OTP 送信 |
| `verifyPublicOtp` | OTP 検証 → アクショントークン発行 |
| `submitPublicMemberUpdate` | allowlist フィルタ済みフィールドを更新 |
| `submitPublicWithdrawalRequest` | 年度末退会申請（WITHDRAWAL_SCHEDULED + 年度末日） |

### 3.3 変更可能フィールド（個人会員のみ）

連絡先（メール・携帯電話）/ 勤務先電話・FAX / 勤務先住所（5フィールド）/ 自宅住所（5フィールド）/ 通知方法 / 郵送先区分

名前・介護支援専門員番号・会員種別の変更は管理者操作のみ（管理画面から）。

### 3.4 フロントエンド

- `src/public-portal/components/MemberUpdateForm.tsx`（4ステップ）
- `src/public-portal/components/WithdrawalRequestForm.tsx`（3ステップ）
- `src/public-portal/App.tsx`: View 型 2 件追加、カード 2 枚追加

### 3.5 退会の仕様

- 既存の `withdrawSelf_` と同じ年度末スケジューリングロジックを踏襲
- `WITHDRAWAL_SCHEDULED` + 退会日 = 当年度末 3/31 を設定
- 退会予定日まで会員マイページへのログイン可能
- 撤回は会員マイページ（ログイン後）から操作

## 6. 操作者確認ポイント

1. 公開ポータル（`/exec?app=public`）のホーム画面に 2 カードが表示される
2. 「会員登録情報を変更する」→ CM番号入力 → OTP メール受信 → コード入力 → フィールド選択 → 値入力 → 完了
3. 「退会を申し込む」→ CM番号入力 → OTP メール受信 → コード入力 → 確認チェック → 完了
4. 変更確認メール・退会確認メールが登録アドレスに届く
5. 無効な CM 番号や誤 OTP でも CM 番号の有無が応答から判別できないこと
