# Release State: v197

Updated: 2026-04-11
Status: current production

## Summary

- Version `197` adds co-located roster/reminder template support and a dialog-based help flow on the roster export console.
- Both fixed deployments were redeployed to `@197` on 2026-04-11.
- Existing roster export behavior remains compatible with legacy `_DATA` and `P_` / `B_` templates.

## Scope

- `backend/Code.gs`
  - Added `REMINDER_TEMPLATE_SS_ID` to system settings read/write/bootstrap paths.
  - Changed roster template data-sheet resolution to prefer `_DATA_ROSTER` and fall back to legacy `_DATA`.
  - Changed roster sheet selection to prefer developer metadata and keep `P_` / `B_` prefix fallback.
  - Added sample template generator `createRosterReminderTemplateExample()`.
- `src/App.tsx`
  - Added system-settings input for `REMINDER_TEMPLATE_SS_ID`.
- `src/components/RosterExport.tsx`
  - Added `テンプレートヘルプを見る` button.
  - Added updated inline guidance for `_DATA_ROSTER` / `_DATA_REMINDER` and metadata fallback rules.
- `src/components/RosterTemplateHelpDialog.tsx`
  - New multi-page modal help UI for template structure, build steps, and settings registration.
- `src/types.ts`
  - Added `reminderTemplateSsId` to `SystemSettings`.

## Compatibility

- Legacy roster templates that only provide `_DATA` still work.
- Legacy roster templates without developer metadata still work through `P_` / `B_` prefix fallback.
- Reminder template support is additive. Existing roster export flows continue to use `ROSTER_TEMPLATE_SS_ID`.
- A single spreadsheet ID can be registered in both `ROSTER_TEMPLATE_SS_ID` and `REMINDER_TEMPLATE_SS_ID`.
