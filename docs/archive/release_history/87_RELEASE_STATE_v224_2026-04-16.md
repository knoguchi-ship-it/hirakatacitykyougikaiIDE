## Summary
### Bulk mail sender: saved templates + safer default attachment setting

Primary files:
- `src/components/BulkMailSender.tsx`
- `src/services/api.ts`
- `backend/Code.gs`

Changes:
- Added bulk-mail template save/load/delete UI for subject and body.
- Added backend APIs for `getBulkMailTemplates`, `saveBulkMailTemplate`, and `deleteBulkMailTemplate`.
- Stored bulk-mail templates in system setting key `BULK_MAIL_TEMPLATES`.
- Changed Drive-folder auto-attach default from ON to OFF to reduce accidental attachment use.
- Kept existing send flow unchanged when the operator explicitly enables auto-attach.

UX / operational impact:
- Operators can reuse vetted message drafts without manually rebuilding subject/body each time.
- Default OFF for auto-attach lowers the chance of unintended file attachment during bulk sends.
- The new template flow follows the same operational pattern already used for credential email templates.
