# リリース記録 v247（2026-04-20）

## 概要

事業所会員の職員入力 UI を `氏 / 名 / セイ / メイ` の分割入力へ統一した。
会員マイページと管理者コンソールの新規職員追加を同じ入力モデルに揃え、
保存時は従来どおり `name` / `kana` を再合成して backend 互換を維持した。

## 実施内容

### `src/components/MemberForm.tsx`

1. 事業所会員の職員欄を `氏 / 名 / セイ / メイ` の 4 入力へ変更
2. 一般職員の自己編集権限を同じ項目単位へ調整
3. 保存前に `name` / `kana` を再合成し、既存 backend API と整合
4. 新規・既存職員の必須/半角ｶﾅ検証を分割項目ベースへ変更

### `src/components/MemberDetailAdmin.tsx`

1. 管理者コンソールの「新規職員追加」を分割入力へ変更
2. 新規ドラフト行の必須/半角ｶﾅ検証を `氏 / 名 / セイ / メイ` 単位へ変更
3. 保存時は `name` / `kana` を再合成して既存データ構造に合わせた

### release

1. `npm run typecheck`
2. `npm run build`
3. `npm run build:gas`
4. `npx clasp push --force`
5. `npx clasp version "v247 split staff name/kana fields in member portal and admin draft UI"`
6. fixed deployment 2 本を `@247` に `redeploy`
