# Release State v226

Updated: 2026-04-17
Version: `226`
Fixed deployments: member `@226` / public `@226`

## Summary

v226 redesigns the individual/support member application form Step 2 ("住所・連絡情報"), merging the former Step 3 "連絡設定" into a single unified step.

## Included Changes

### 1. Step Structure Change (INDIVIDUAL / SUPPORT)

| Before | After |
|---|---|
| ①会員種別 ②基本情報 ③住所情報 ④連絡設定 ⑤入力確認 | ①会員種別 ②基本情報 ③住所・連絡情報 ④入力確認 |

Step 3 "連絡設定" is removed. All contact fields are now in Step 2.

### 2. 郵送先ラジオカード選択

- "🏢 勤務先 / 🏠 自宅" radio-card selector placed at the top of Step 2.
- The selected section receives a blue highlight border and a "郵送先・必須" badge.
- The non-selected section is optional ("任意" badge) but its data is still saved if filled.

### 3. 事業所名を任意項目に変更

- `officeName` is now an independent optional field in the 勤務先 section.
- It is no longer required even when 勤務先 is selected as the mailing destination.
- Rationale: users should not be forced to enter 事業所名 just to fill their mailing address; the DB benefits from having it when available.

### 4. 電話番号 OR 携帯電話番号

- Either `phone` OR `mobilePhone` being filled satisfies the contact requirement.
- Both fields show "どちらか一方必須" amber badge.
- Composite error `_phone` is cleared when either field changes.

### 5. メールアドレスを住所ステップに統合

- `email` field moved from removed Step 3 into the 連絡先 section of Step 2.
- Displayed in a prominent blue gradient card with note "📨 ログイン情報・各種お知らせの送付先です".

### 6. 発送方法ラジオカード

- "📧 メール配信 / ✉️ 郵送希望" also converted to radio-card format consistent with 郵送先 selector.

### 7. バリデーション変更

- `validateAddress` + `validateContact` → unified `validateAddressAndContact`
- Required address fields are determined by `preferredMailDestination`, not by what was filled.
- `officeName` removed from required fields.
- Phone: `phone || mobilePhone` (OR logic).
- Email: validated in Step 2 (was Step 3).

### 8. アクセシビリティ対応

- Radio card groups use `role="radiogroup"` and `aria-label`.
- Required inputs carry `aria-required` and `aria-invalid` attributes.
- Error messages use `role="alert"`.
- Focus ring on radio card buttons.

### 9. レスポンシブ

- All grid sections collapse to single-column on narrow viewports (`sm:grid-cols-2`).
- Confirmation table wraps in `overflow-x-auto` for mobile.
