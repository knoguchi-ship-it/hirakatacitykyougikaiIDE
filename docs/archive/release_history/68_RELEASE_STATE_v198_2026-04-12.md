# Release State: v198

Updated: 2026-04-12
Status: current production

## Summary

- Version `198` replaces the roster-export long inline guide with a short quick-start flow and a dedicated help page.
- System settings now support template validation before save for roster and reminder spreadsheets.
- Existing roster export behavior remains compatible with legacy `_DATA` and `P_` / `B_` templates.

## Scope

- `backend/Code.gs`
  - Added `validateTemplateSpreadsheet_()`.
  - Added spreadsheet URL-to-ID normalization in `updateSystemSettings_()`.
  - Added API action permission and dispatch for `validateTemplateSpreadsheet`.
- `src/App.tsx`
  - Added `template-help` view.
  - Wired roster export to open the dedicated help page.
  - Added `TemplateValidationPanel` to system settings.
- `src/components/RosterExport.tsx`
  - Replaced long inline template explanation with a quick-start guidance card.
  - Added explicit links to the dedicated help page and system settings.
- `src/components/TemplateHelpPage.tsx`
  - New dedicated help page for template preparation and operations.
- `src/components/TemplateValidationPanel.tsx`
  - New validation UI for roster/reminder spreadsheet IDs or URLs.
- `src/services/api.ts`
  - Added `validateTemplateSpreadsheet()` client API.
- `src/shared/types.ts`
  - Added template validation result types.

## Compatibility

- Legacy roster templates that only provide `_DATA` still work.
- Legacy roster templates without developer metadata still work through `P_` / `B_` prefix fallback.
- Reminder template support remains additive.
- Existing ZIP generation logic and member filtering behavior are unchanged.
