## 3. 実装詳細

### 3.1 公開入会申込

- `officeNumber` は `^[A-Za-z0-9]{10}$` の allowlist へ変更。
- 入力欄は `maxLength=10` とし、半角英数字10文字のサポートテキストを表示。
- submit payload でも同一ルールを適用し、フロント回避では保存できないよう統一。

### 3.2 事業所職員登録

- 同一事業所内でのメールアドレス重複禁止を撤廃。
- `lastName / firstName / lastKana / firstKana / careManagerNumber / email` がすべて空の職員カードは、未入力行として除外。
- 一部でも入力がある職員カードは従来どおり必須入力と整合チェックを行う。

### 3.3 データ管理コンソール

- 削除操作は論理削除へ変更。
- 更新対象:
  - `T_会員`: `WITHDRAWN` + `削除フラグ=true`
  - `T_事業所職員`: `LEFT` + `削除フラグ=true`
  - `T_認証アカウント`: `アカウント有効フラグ=false` + `削除フラグ=true`
  - `T_管理者Googleホワイトリスト`: `有効フラグ=false` + `削除フラグ=true`
- 保持対象:
  - `T_ログイン履歴`
  - `T_研修申込`
  - `T_年会費納入履歴`
  - `T_年会費更新履歴`
- `T_削除ログ` は操作前スナップショット記録として継続利用。
