# リリース記録 v248（2026-04-20）

## 概要

第三者評価（`docs/109_THIRD_PARTY_ASSESSMENT_2026-04-20.md`）で指摘された Critical/High 所見のうち、
認可境界と IDOR に関わる最重要部分を修正した。

## 修正内容

### backend/Code.gs

#### 1. ADMIN_LOGIN_ACTIONS 新設（管理者ログイン修正）

`checkAdminBySession` / `adminLoginWithData` を `ADMIN_LOGIN_ACTIONS` に登録。
これらは `Session.getActiveUser()` によって自己完結する認証アクションのため、
事前の admin session 検証を必要とせず、独立したレジストリで管理する。

前回の差分で deny-by-default チェックが導入された際に、これらのアクションが
どのレジストリにも含まれなくなり、管理者ログインが broken になっていた問題を修正。

#### 2. ADMIN_ACTION_PERMISSIONS に不足アクションを追加

以下7アクションがレジストリ未登録で管理者から到達不能になっていた問題を修正。

| アクション | 追加権限 |
|---|---|
| `fetchAllData` | MASTER/ADMIN/TRAINING_MANAGER/TRAINING_REGISTRAR |
| `initRosterExport` | MASTER/ADMIN |
| `processRosterChunk` | MASTER/ADMIN |
| `finalizeRosterExport` | MASTER/ADMIN |
| `cleanupRosterExport` | MASTER/ADMIN |
| `generateMailingListExcel` | MASTER/ADMIN |

#### 3. 会員セッショントークン検証ゲート（IDOR 修正）

`processApiRequest` に `CacheService` ベースのセッション検証ゲートを追加。

- ログイン以外の MEMBER_ALLOWED_ACTIONS は `sessionToken` の提示が必須
- CacheService から sessionToken を照合し、存在しない場合は `member_session_expired` で拒否
- 照合成功時、`parsedPayload.loginId / memberId / staffId` をサーバー検証済み値で上書き
- これにより、クライアント申告の識別子を信頼しない設計に移行（IDOR を根絶）

#### 4. memberLogin_ にセッショントークン発行を追加

認証成功時に UUID を生成し、`CacheService.getScriptCache()` に TTL 1800秒（30分）で保存。
返却値に `sessionToken` を追加。

### src/services/api.ts

- `MemberLoginResult` に `sessionToken: string` を追加
- `ApiClient` インターフェースに `setMemberSessionToken(token)` を追加
- `GasApiClient` に `memberSessionToken` 状態と `memberSessionPayload()` ヘルパーを追加
- 以下7メソッドに `...this.memberSessionPayload()` を追加:
  - `getMemberPortalData`, `updateMemberSelf`, `changePassword`
  - `applyTraining`, `cancelTraining`, `withdrawSelf`, `cancelWithdrawalSelf`

### src/App.tsx

- `applyAuthContext` の ctx 型に `sessionToken?: string` を追加
- 会員ログイン（`authMethod === 'PASSWORD'`）成功時に `api.setMemberSessionToken(ctx.sessionToken)` を呼び出し
- ログアウト時に `api.setMemberSessionToken(null)` でクリア

### docs/113_TASK_CM_NUMBER_EDIT_POLICY_2026-04-20.md（新規）

会員マイページでの介護支援専門員番号編集ポリシーを次フェーズの課題として記録。
