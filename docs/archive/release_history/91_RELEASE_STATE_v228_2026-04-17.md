# Release State v228

Updated: 2026-04-17
Version: `228`
Fixed deployments: member `@228` / public `@228`

## Summary

v228 improves the public membership application flow by separating workplace affiliation from mailing-address input, fixing conditional address validation, and adding an admin toggle to hide login information on the completion screen until the member portal is ready.

## Included Changes

### 1. Completion screen login-info visibility control

- Added `T_システム設定.PUBLIC_PORTAL_COMPLETION_LOGIN_INFO_VISIBLE`.
- Admins can now switch the completion screen login-information block on or off without code changes.
- When the block is hidden, the screen shows guidance to check email instead of exposing the login ID on screen.

### 2. Completion messaging cleanup

- Added a dedicated `今後のご案内` block on the completion screen.
- Added a message telling the applicant to check the registered email address for annual-fee and bank-transfer guidance.
- The completion screen now varies its login-information guidance depending on whether credential email sending is enabled.

### 3. Address-form validation fixes

- Removed the non-selected-side postcode false-positive validation path caused by default placeholder values.
- When `勤務先` is selected as the mailing destination, `事業所名` is now required.
- Empty home postcode no longer blocks progress when the mailing destination is workplace.

### 4. Workplace affiliation vs mailing address separation

- Split `勤務先・所属情報` from `勤務先住所` in the personal/support member flow.
- When workplace is the mailing destination, the entered office name is shown as the recipient office name inside the workplace-address card.
- Confirmation view labels were adjusted so workplace affiliation and workplace address are easier to distinguish.
