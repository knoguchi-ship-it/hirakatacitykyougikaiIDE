# Release State v229

Updated: 2026-04-17
Version: `229`
Fixed deployments: member `@229` / public `@229`

## Summary

v229 fixes two member-portal annual-fee regressions: the previous fiscal year could disappear when the current fiscal year was auto-supplemented as unpaid, and the unpaid guidance dialog could fail to show the common transfer account when record-level account data was incomplete.

## Included Changes

### 1. Annual-fee history display fix

- Adjusted member-page `annualFeeHistory` construction so the current fiscal year unpaid supplement no longer pushes out the prior fiscal year record.
- The member page now keeps up to two stored historical records and, when needed, prepends the current fiscal year unpaid supplement.
- This ensures 2025年度 remains visible even when 2026年度 is supplemented as unpaid.

### 2. Transfer-account fallback fix

- Tightened the member-page transfer-account selection logic.
- If the annual-fee record contains missing or partial account data, the UI now falls back to the common `ANNUAL_FEE_TRANSFER_ACCOUNT` setting instead of treating the incomplete record as authoritative.
- The unpaid guidance dialog therefore continues to show bank details for unpaid members.
