## 変更内容

### member ログイン画面から管理者ログイン UI を除去

**目的**: 会員マイページに管理者ログインタブ・Google 管理者ログイン仕様が表示される状態を解消する。

**修正内容**:
- `src/App.tsx` の未認証画面を shell mode 別に明示分岐。
- member shell では会員ログインフォームのみ表示。
- admin shell では Google 管理者ログインのみ表示。
- integrated shell の既存タブ式ログインは互換目的で維持。

### app 別 action registry への移行

**目的**: v279 の分類ベース gate を、アプリ別 allowlist に寄せ、境界外 action が評価対象に入らない構造へ近づける。

**修正内容**:
- `getActionRegistryForCurrentApp_()` を追加。
- public は `PUBLIC_ALLOWED_ACTIONS` のみ有効。
- member は `MEMBER_ALLOWED_ACTIONS` のみ有効。
- admin は `ADMIN_LOGIN_ACTIONS` と `ADMIN_ACTION_PERMISSIONS` のみ有効。
- member から `adminLoginWithData` などの admin action は `unsupported_action` で拒否。

### member/admin 生成スクリプトの境界固定

**目的**: 将来 `build:gas:member` / `build:gas:admin` を実行した際に、`APP_SECURITY_BOUNDARY` が `public` に巻き戻る事故を防ぐ。

**修正内容**:
- `scripts/build-member-gas.mjs` は `backend/Code.gs` をコピー後、境界値を `member` に置換して出力。
- `scripts/build-admin-gas.mjs` は `backend/Code.gs` をコピー後、境界値を `admin` に置換して出力。

## 現時点の分離レベル

今回の変更は **Phase 2: app 別 action registry と member UI 分離**。

- 会員ログイン画面から管理者ログイン UI は除去済み。
- member API は member allowlist のみを参照。
- admin API は admin allowlist のみを参照。
- public API は public allowlist のみを参照。
- 関数本体の dead code 削除、manifest / scope 最小化は次フェーズで実施する。
