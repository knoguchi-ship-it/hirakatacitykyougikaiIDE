# Release State: v209

Date: 2026-04-16
Version: `209`
Fixed deployments: member `@209` / public `@209`

## Summary

公開ポータルからの入会申込時に、ログイン情報（ID/パスワード）メールを
送信するかどうかを管理者が設定できるようにした。
あわせてメール件名・本文もカスタマイズ可能にし、マージタグで差し込みに対応。

## Changes

### GAS backend (`backend/Code.gs`)

**新定数**
- `MEMBER_PORTAL_URL`: 会員マイページ固定 URL（メール本文デフォルトで使用）
- `CREDENTIAL_EMAIL_DEFAULT_SUBJECT`: デフォルト件名
- `CREDENTIAL_EMAIL_DEFAULT_BODY`: デフォルト本文（マージタグ入り）

**`sendCredentialEmail_(toEmail, loginId, password, memberName, opts)`**
- `opts.subject` / `opts.body` でテンプレートを受け取り、マージタグを置換して送信
- マージタグ: `{{氏名}}` `{{ログインID}}` `{{パスワード}}` `{{会員マイページURL}}`
- `opts` 省略時はデフォルトテンプレートを使用（後方互換）
- 旧実装の URL ハードコード (`AKfycbycE2_...`) を正しい固定 deployment URL に修正

**`submitMemberApplication_(payload)`**
- 処理開始時に `CREDENTIAL_EMAIL_ENABLED` / `SUBJECT` / `BODY` を T_システム設定 から一度だけ読み込む
- `CREDENTIAL_EMAIL_ENABLED = 'false'` の場合はメール送信をスキップ（事業所・個人・賛助 すべて）
- `sendCredentialEmail_` に `opts` を渡すよう変更

**`getSystemSettings_()`**
- `credentialEmailEnabled`: bool（未設定時は `true`、後方互換）
- `credentialEmailSubject`: string
- `credentialEmailBody`: string

**`updateSystemSettings_(request, callerPermLevel)`**
- 上記3キーを MASTER/ADMIN 共通で保存可能
- 件名・本文が空の場合はデフォルトに差し戻す

**`insertSystemSettingKeysForV209()`**
- ワンタイム実行済み: 3キーを T_システム設定 に追加
  - `CREDENTIAL_EMAIL_ENABLED = 'true'`
  - `CREDENTIAL_EMAIL_SUBJECT` = デフォルト件名
  - `CREDENTIAL_EMAIL_BODY` = デフォルト本文

### TypeScript types (`src/types.ts`)
- `SystemSettings.credentialEmailEnabled?: boolean`
- `SystemSettings.credentialEmailSubject?: string`
- `SystemSettings.credentialEmailBody?: string`

### Frontend (`src/App.tsx`)
- 設定画面に「入会時ログイン情報メール設定」セクションを追加
  - ON/OFF トグルスイッチ（OFF 時に amber 注意バナーを表示）
  - メール件名テキスト入力 + デフォルトに戻すボタン
  - メール本文テキストエリア（等幅フォント・リサイズ可）+ デフォルトに戻すボタン
  - マージタグ使用可能の説明を表示
- `applySystemSettings` に3フィールドの初期化を追加
- `updateSystemSettings` 呼び出しに3フィールドを追加

## 初回利用手順

1. 管理者コンソール → 設定 → 「入会時ログイン情報メール設定」
2. ON/OFF を必要に応じて変更
3. 件名・本文を編集（マージタグを活用）
4. 「設定を保存」ボタンで保存
