## 7. GAS 開発の落とし穴（経験済みのバグ）

### (1) `setValues()` + `UrlFetchApp` は必ず `flush()` を挟む

GAS の `setValues()` は書き込みをバッファリングする。  
`UrlFetchApp.fetch()` で Spreadsheet を HTTP エクスポートする際、  
バッファが未フラッシュだと古い状態がエクスポートされる（v208 で実際に踏んだ）。

```javascript
// NG: バッファが残ったままエクスポートされる
sheet.getRange(...).setValues(rows);
UrlFetchApp.fetch(exportUrl, ...);

// OK: 必ず flush を挟む
sheet.getRange(...).setValues(rows);
SpreadsheetApp.flush();              // ← これが必須
UrlFetchApp.fetch(exportUrl, ...);
```

### (2) `clasp deploy` は絶対に禁止

`clasp deploy`（`--redeployId` なし）は新しい Deployment ID を生成し、本番 URL が変わる。  
固定 URL の運用が崩壊する。リリースは必ず `clasp redeploy <固定ID>` のみ使う。

### (3) `forEach` 内の `return` は外側の関数を抜けない

GAS（JavaScript）の `forEach` コールバック内の `return` はそのコールバックを抜けるだけ。  
外側の関数を早期終了したい場合は `Array.prototype.some()` か `for` ループを使う。

### (4) `clasp run` は devMode = true でのみ動作

`clasp run --nondev`（本番 deployment に対してのみ実行）は失敗する。  
`clasp run` は開発モード限定と割り切って使う。本番動作確認はブラウザで行う。

### (5) OWASP Mass Assignment — フィールドを追加したら allowlist を忘れずに更新

新しいフィールドを追加した場合、以下の allowlist にも追加しないと書き込みが無視される:

```javascript
// backend/Code.gs
MEMBER_WRITABLE_FIELDS_       // 会員自身が書き込める項目
ADMIN_MEMBER_WRITABLE_FIELDS_ // 管理者が書き込める項目
```

## 9. 認証の仕組み

### 会員ログイン
- ログインID（介護支援専門員番号）+ パスワード
- `T_認証` テーブルで照合、`bcrypt` 相当のハッシュ比較

### 管理者ログイン
- Google アカウントで GAS ページを開くと `Session.getActiveUser().getEmail()` で Google メールを取得
- `T_管理者Googleホワイトリスト` のメールアドレスと照合
- 一致すれば権限レベルを返す（GIS は廃止済み）
- ホワイトリスト・認証行は `CacheService.getScriptCache()`（TTL 300秒）にキャッシュ

## 12. 重要な禁止事項まとめ

| 禁止 | 理由 |
|---|---|
| `clasp deploy`（全形式） | 新 Deployment ID 生成 → URL 変更 |
| `clasp redeploy` 片方だけ | 会員 / 公開 の両 deployment が常に同一バージョンを指すこと |
| `npx clasp run seedDemoData` | 本番DB全削除。実行前に明示的な許可が必要 |
| `--no-verify` でのコミット | フック bypassing |
| コミット対象の無限定ステージング | `.env` や無関係ファイルを巻き込まないよう `git add <ファイル名>` で限定 |
| Demo / Mock フローの復活 | 廃止済み。再実装禁止 |
| 会員ページへの管理者固有設定追加 | ビジネスルール上の禁止（BUSINESS会員への配送設定追加など） |
