# Google Apps Script クォータ・制限一覧

作成日: 2026-03-31
対象: 枚方市介護支援専門員連絡協議会 会員システム
位置づけ: 公式クォータの全文転記ではなく、この案件で監視すべき制限と確認観点をまとめた運用メモ

---

## 1. 運用前提

- Apps Script の日次クォータと各種制限は、ユーザー種別・契約形態・関連プロダクトの制限に依存する。
- 公式ドキュメント上、クォータは「最初のリクエストから24時間後にリセット」され、「予告なく変更される場合がある」。
- したがって本書では、変動しやすい日次上限値の断定よりも、この案件で監視すべき制限と実装上の安全マージンを整理する。
- 正式な上限値を要する判断の前には、必ず公式の `Apps Script Quotas` を当日再確認する。

## 2. この案件で直接効く制限

| 観点 | 公式・実装前提 | 本システムの状況 |
|------|---------------|----------------|
| スクリプト実行時間（1回） | Apps Script は実行時間制限がある。重い読込処理は分割・キャッシュ前提で設計する | `fetchAllDataFromDb_()` が最大。キャッシュヒット時 < 1秒、ミス時 3-5秒 |
| CacheService 1キーサイズ | 1キーあたり 100KB 制限を前提にする | 90KB chunk で分割保存し 10% マージンを確保 |
| CacheService TTL | 公式上限以内で設定する | 300秒。書き込み時 invalidation 実装済み |
| メール送信日次枠 | 日次枠は契約形態で変動しうるため、送信前に公式値確認が必要 | 会員数 ~200 規模では通常運用で余裕あり。複数研修の同日一斉送信時のみ注意 |
| `google.script.run` / HTML 応答サイズ | HTML Service とブラウザ体感の両面で監視が必要 | 会員ポータル ~513KB、公開ポータル ~331KB |
| Spreadsheet 読込量 | API 単位で全件読込すると実行時間に効く | `getValues()` / `setValues()` のバッチ操作で対応 |

## 3. CacheService

| 制限項目 | 上限/前提 | 本システムの状況 |
|---------|------|----------------|
| 1キーあたりの値サイズ | 100KB | チャンクサイズ 90KB（10%マージン）で分割保存 |
| キャッシュ合計サイズ | ドキュメント化されていない | `putAll` でバッチ書き込み。画面データ合計 ~500KB |
| TTL | 最大 21,600秒（6時間） | 300秒に設定。書き込み時 invalidation 実装済み |
| `put`/`get` 呼び出し回数 | ドキュメント化されていない | 通常利用で問題なし |

## 4. SpreadsheetApp / HTML Service

| 制限項目 | 上限/前提 | 本システムの状況 |
|---------|------|----------------|
| セル数（1スプレッドシート） | 10,000,000セル | 現在 ~20,000セル。十分な余裕 |
| シート数 | 200シート | 現在 ~15シート |
| 1回の `getValues()` | 制限なし（実行時間制限に依存） | `getDataRange().getValues()` で全行取得 |
| HTML レスポンスサイズ | 約5MB | 会員ポータル ~513KB、公開ポータル ~331KB。余裕あり |
| 同時接続数 | ドキュメント化されていない | 小規模（~200会員）で問題なし |
| `google.script.run` レスポンスサイズ | ドキュメント化されていない（実測 ~5MB） | キャッシュ済みデータ ~500KB |

## 5. UrlFetch / DriveApp / 外部依存

| 制限項目 | 上限/前提 | 本システムの状況 |
|---------|--------------|----------------|
| `UrlFetchApp.fetch`（日次） | 公式値は契約条件で変動しうるため要再確認 | 現在は外部API呼び出しなし |
| UrlFetch レスポンスサイズ | 公式値確認が必要 | 現在は未使用 |
| ファイル作成（1日） | ドキュメント化されていない | 研修案内状アップロード。1日数件 |
| ストレージ | Workspace プラン依存 | 案内状 PDF のみ |

## 6. 監視ポイント

本システムの規模（会員 ~200名、管理者 ~5名）では、上記クォータの超過リスクは極めて低い。ただし、以下の状況では注意が必要:

1. **会員数が 1,000 を超えた場合**: `fetchAllDataFromDb_()` の実行時間とキャッシュサイズを監視
2. **一斉メール送信**: リマインダーを複数研修で同日に送信する場合、当日の公式メール送信枠を確認する
3. **HTML バンドルサイズ**: 機能追加に伴い 1MB を超えると体感パフォーマンスに影響（P-03）
4. **GCP 移行後**: Cloud Run / Firestore に移行するとこれらの制限は解消されるが、GCP 側の無料枠・課金体系に注意
