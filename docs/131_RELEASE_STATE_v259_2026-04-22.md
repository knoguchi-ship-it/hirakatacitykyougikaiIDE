対象: 入会通知メールの送信元アドレス設定・エイリアス選択

## 1. サマリ

- 管理画面「入会通知メール」に **送信元メールアドレス** の選択欄を追加。
- 選択肢は **主メールアドレス + Gmail send-as alias** に限定し、自由入力を許容しない構成へ変更。
- backend は **保存時・送信時の両方** で alias 妥当性を検証し、既存メール送信フローへ安全に統合。

## 3. 実装詳細

### 3.1 設定 UI

- 「入会通知メール」セクションに `送信元メールアドレス` セレクトを追加。
- alias 一覧は `getAdminEmailAliases()` を利用して取得し、再取得ボタンを設置。
- alias 取得に失敗した場合は警告を表示し、保存済み値は選択肢として維持する。

### 3.2 backend 検証

- `T_システム設定.CREDENTIAL_EMAIL_FROM` を追加。
- 保存時は `validateRequestedFromAddress_()` で、実行アカウントの主メールアドレスまたは Gmail send-as alias のみ許可。
- 実送信は `sendEmailWithValidatedFrom_()` を経由し、送信直前にも同一条件で再検証。

### 3.3 既存機能への影響抑制

- 送信元は自由入力ではなく allowlist 選択のみ。
- 主メールアドレス選択時は既存運用と同等の挙動を維持。
- alias 選択時のみ Gmail send-as 指定を適用し、他メール機能と同じ共通ヘルパーへ統一。
