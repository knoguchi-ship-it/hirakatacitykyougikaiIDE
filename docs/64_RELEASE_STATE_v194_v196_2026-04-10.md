### GAS 実装詳細

#### `getMembersForBulkMail_(payload)`

- フィルタ: `memberTypes`（INDIVIDUAL/BUSINESS/SUPPORT 複数可）、`memberStatus`（ACTIVE/INCLUDING_SCHEDULED/ALL）、`staffStatus`（ACTIVE/ALL）、`mailingFilter`（OPT_IN/ALL）、`excludeNoEmail`（bool）
- INDIVIDUAL/SUPPORT: `T_会員.代表メールアドレス`、`mailingFilter` 適用なし（アドレスがあれば送信）
- BUSINESS: `T_事業所職員`（メール配信希望コード ≠ 'NO' でフィルタ）
- 返却型: `BulkMailRecipient[]`

#### `sendBulkMemberMail_(payload)`

- サーバー側で全宛先を再フェッチ（フロント送信リストを信頼しないセキュリティ設計）
- `from` エイリアスを `GmailApp.getAliases()` で検証
- Drive自動添付: `BULK_MAIL_AUTO_ATTACH_FOLDER_ID` フォルダ内を検索し、ファイル名に `姓名`（スペースなし）が含まれる先頭1件を添付
- 送信後に `T_メール送信ログ` へ追記（個人情報は記録しない。件名テンプレート・件数のみ）
- 返却: `{ sent, total, errors, autoAttachMissed, logId }`

#### `getEmailSendLog_(payload)`

- `T_システム設定.EMAIL_LOG_VIEWER_ROLE` を実行時に動的読み込みして権限チェック
- 返却: `EmailSendLog[]`（新しい順）

### フロントエンド機能

- フィルタパネル（種別チェック・在籍状態・メール配信希望・メール未登録除外）
- 宛先一覧（`selectedIds: Set<string> | null` = null で全選択パターン、excludedIds で部分解除）
- メール配信希望コード='NO' の職員がリストに含まれる場合に ⚠ 警告表示
- 差し込みタグエディタ（`{{氏名}}` `{{事業所名}}` `{{会員番号}}`、カーソル位置挿入）
- エイリアスセレクト・共通添付・Drive自動添付 ON/OFF・個人別添付
- 送信確認ダイアログ（宛先数・内容確認）
- 送信結果表示（成功数・失敗・autoAttachMissed 一覧）
- 送信ログ閲覧（EMAIL_LOG_VIEWER_ROLE 権限チェック）

| `backend/index.html` / `index_public.html` | ビルド成果物更新 |

### GAS 実装詳細

#### `getMembersForRoster_(payload)`

- フィルタ: `memberTypes`（複数可）、`memberStatus`（ACTIVE/INCLUDING_SCHEDULED/ALL）、`annualFeeStatus`（ALL/UNPAID/PAID）、`annualFeeYear`（int）
- 年会費ステータス UNPAID: T_年会費納入履歴にレコードなし（NONE）も未納扱い
- BUSINESS 会員は `enrolledStaffCount`（在籍職員数）を付加
- 返却型: `RosterTarget[]`

#### `generateRosterZip_(payload)`

- `memberIds: string[]` で対象を受け取る（50件超はエラー返却）
- テンプレートSS（`ROSTER_TEMPLATE_SS_ID`）を DriveApp.makeCopy() で一時コピー（1バッチ1コピー）
- `_DATA` シートを自動作成・非表示化
- 各会員ループ:
  1. `_DATA` シートに Row1（ヘッダー）+ Row2（データ）+ Row4（職員ヘッダー）+ Row5+（職員データ）を書き込み
  2. `SpreadsheetApp.flush()` → 表示シートのセルを読み取り（数式強制再計算）
  3. `UrlFetchApp.fetch(exportUrl, {headers: {Authorization: 'Bearer ' + ScriptApp.getOAuthToken()}})` でPDF取得
  4. blob を配列に蓄積
- 全blob → `Utilities.zip()` → `DriveApp.createFile()` → DLリンク返却
- 一時コピーは処理後即削除
- 職員ソート: REPRESENTATIVE → ADMIN → STAFF（ENROLLED のみ）

#### `_DATA` シート列マッピング（Row1/2）

| 列 | フィールド |
|---|---|
| A | 会員番号 |
| B | 会員種別 |
| C | 姓 |
| D | 名 |
| E | フリガナ姓 |
| F | フリガナ名 |
| G | 勤務先名 |
| H | 事業所番号 |
| I | 勤務先郵便番号 |
| J | 勤務先都道府県 |
| K | 勤務先市区町村 |
| L | 勤務先住所 |
| M | 勤務先電話 |
| N | 勤務先FAX |
| O | 自宅郵便番号 |
| P | 自宅都道府県 |
| Q | 自宅市区町村 |
| R | 自宅住所 |
| S | メールアドレス |
| T | 入会日 |
| U | 年会費状態 |
| V | 年会費年度 |
| W | 介護支援専門員番号 |

Row4（職員ヘッダー）/ Row5+（職員データ）: A=職員番号, B=職員権限, C=姓, D=名, E=フリガナ姓, F=フリガナ名, G=メールアドレス, H=入会日, I=職員状態

### フロントエンド機能

- フィルタパネル（種別チェック INDIVIDUAL/BUSINESS/SUPPORT・在籍状態・年会費ステータス・年度）
- 対象一覧（`selectedIds: Set<string> | null` null=全選択パターン・種別ごと全選択ボタン・50件超 overLimit 警告）
- BUSINESS 会員には在籍職員数を（N名在籍）として表示
- ZIP生成スピナー（所要時間目安表示）
- DLリンク（`https://drive.google.com/uc?export=download&id={fileId}`）
- エラー一覧表示
- `_DATA` シート列マッピングのインラインガイド（折りたたみ可）

### 前提条件・運用注意

- **システム設定画面で `ROSTER_TEMPLATE_SS_ID` を登録すること**。未登録時はコンソールに警告が表示され生成ボタンが無効化される。
- テンプレートSS側では `_DATA` シート（Row1: ヘッダー行、Row2: データ行）を参照するよう各表示シートに数式を記述すること（例: `=IFERROR(TRIM(_DATA!C2&" "&_DATA!D2), "")`）。
- テンプレート構成の詳細ガイドは `docs/66_ROSTER_TEMPLATE_GUIDE_2026-04-10.md` を参照。
- 生成されたZIPファイルはDriveに一時保存される。DL後は手動削除を推奨。
- 50件/バッチ制限（GAS 6分タイムアウト対策）。超過時はフロント側でエラー表示し、件数を絞って再実行するようガイドする。
