# リリース記録 v246（2026-04-19）

## 概要

会員マイページ保存時の `loginId` 解決を認証済みコンテキスト基準に揃え、
事業所会員情報更新の未解決課題に対する第一段の本番修正を反映した。
あわせて Apps Script の古い version を整理し、最新 20 件だけを残す運用状態へ整えた。

## 背景

`updateMemberSelf_()` は `payload.loginId` を前提に認証アカウントを引く実装だが、
フロントエンドの保存経路ではログインフォーム入力値の `memberLoginId` を優先して渡していた。
再読込系は `authenticatedContext.memberPortalLoginId` を基準にしており、
保存経路と読込経路のアンカーが一致していなかった。

また、GAS version 上限に対する予防として、デプロイ前に直近 20 件以外の version を削除した。

## 実施内容

### `src/App.tsx`

1. `handleMemberSave()` の会員保存経路で `authenticatedContext.memberPortalLoginId || memberLoginId` を使用
2. `memberPortalLoginId` を解決できない場合は保存を継続せずエラー化
3. `MemberForm` へ渡す `loginId` も同じ解決ルールに統一

### Apps Script version 整理

1. デプロイ前に Project History の bulk delete で古い version を削除
2. `v246` 作成後に旧 `226` を追加で削除し、`227`〜`246` の 20 件だけを保持

### release

1. `build:gas`
2. `npx clasp push --force`
3. `npx clasp version "v246 member self-save loginId anchor fix + handover alignment"`
4. fixed deployment 2 本を `@246` に `redeploy`
