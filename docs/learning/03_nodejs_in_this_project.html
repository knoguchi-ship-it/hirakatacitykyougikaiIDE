この案件で Node.js がしていること

この案件では、Node.js は本番アプリの表示基盤ではありません。 本番でユーザーに画面を見せるのはブラウザと Google Apps Script です。 Node.js はその前段で、ビルド・型チェック・作業コマンド実行を担当しています。

1

依存関係を入れる

npm install で React、Vite、TypeScript などを使えるようにします。

2

型チェックする

npm run typecheck で TypeScript の整合性を確認します。

3

画面をビルドする

npm run build や npm run build:gas で配布用HTMLを作ります。

4

GAS へ渡す準備をする

生成したHTMLを backend/index.html などに配置し、Apps Script 側へ反映します。

誤解しやすい点: Node.js とローカルサーバーは同義ではない

この案件では npm run dev を使うとローカルでブラウザ確認できますが、 それは Node.js 自体が画面を出しているのではなく、 Node.js 上で Vite 開発サーバーが動いているからです。

  • Node.js: JavaScript をブラウザ外で動かす実行環境
  • Vite: Node.js 上で動く開発ツール。開発サーバー機能も持つ
  • ブラウザ: 実際に画面を描画し、React のコードを実行する場所

つまり流れは、 Node.js が Vite を動かす → Vite がローカルサーバーを提供する → ブラウザがそこへアクセスして画面を見る です。

実際に Node.js が関わるファイル

  • package.json: 実行コマンド定義
  • scripts/build-all.mjs: 会員/公開ポータルの順次ビルド
  • scripts/build-gas.mjs: ビルド成果物を GAS 用にコピー
  • vite.config.ts: Vite ビルド設定
  • tsconfig.json: TypeScript 設定

Node.js が直接やっていないこと

  • 本番ユーザー向け画面の配信
  • Google Apps Script の API 実行そのもの
  • Spreadsheet を本番 DB として操作すること
  • 管理者認証や会員認証の本番判定
npm run typecheck
Node.js 上で TypeScript コンパイラを動かし、コードの型の矛盾を検出します。
npm run build
Node.js 上で Vite を動かし、React と TypeScript のコードをブラウザで動く形にまとめます。
npm run build:gas
Node.js 上で会員ポータルと公開ポータルの HTML を生成し、GAS 配置用ファイルへコピーします。
重要な事実
この案件では、Node.js がないとローカルのビルドや型チェックは困難ですが、 Node.js だけでは本番動作確認は完結しません。最終確認は GAS 上で行う必要があります。

つまりこの案件における Node.js は、「本番の心臓部」ではなく「開発と出荷準備の作業台」です。 コードを整え、まとめ、GAS に渡す準備をする役割を担っています。
