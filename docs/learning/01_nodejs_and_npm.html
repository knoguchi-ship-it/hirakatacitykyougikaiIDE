Node.js と npm の違い

この2つは一緒に語られがちですが、役割は同じではありません。 短く言うと、Node.js は JavaScript を実行する環境、 npm はパッケージ管理とコマンド実行の仕組みです。

Node.js

Node.js とは

ブラウザの外で JavaScript を動かすための実行環境です。 JavaScript のコードを、Windows や macOS などの OS 上で動かせます。

  • JavaScript を実行する
  • ファイルを読む、書く
  • 他のプログラムを起動する
  • 開発用ツールを動かす
npm

npm とは

Node.js まわりのライブラリを入れたり、プロジェクトで決めた作業コマンドを実行したりする仕組みです。 一般に Node.js と一緒に使います。

  • 依存ライブラリをインストールする
  • バージョンを固定しやすくする
  • npm run build のように定型作業を実行する

正確に言うと、「Node.js はローカルで JavaScript を動かすためのツール」という理解は大筋で正しいです。 ただしより正確には、「JavaScript をブラウザ外で実行する環境」が Node.js です。 npm はその上で使う管理・実行の仕組みです。

重要な補足

Node.js 自体が必ずローカルサーバーになるわけではない

ここが誤解されやすい点です。Node.js は「JavaScript を実行する環境」ですが、 Node.js を入れただけで自動的にローカルサーバーが立つわけではありません。

  • node sample.js なら、単に JavaScript の処理を1回実行して終わることがあります。
  • ローカルサーバーになるのは、Node.js 上でサーバー用プログラムを動かしたときです。
  • この案件では、その代表例が npm run dev で起動する Vite 開発サーバーです。

つまり、あなたの理解を正確に言い直すと、 「Node.js は JavaScript をローカルで動かす実行環境であり、 その上で Vite などの開発用サーバーを動かすことで、ブラウザ確認ができることがある」 となります。
