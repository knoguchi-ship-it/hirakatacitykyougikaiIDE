デプロイ

🚫
clasp deploy は全形式禁止。 新しいIDが生成されURLが変わる。版更新は clasp redeploy のみ使用。

標準リリース手順

npm run build:gas
# HTML を各 split にコピー
cp backend/index_admin.html gas/admin/index.html
cp backend/index.html gas/member/index.html

# 3プロジェクトへ push
npx clasp push --force
npx clasp push --force -P gas/admin
npx clasp push --force -P gas/member

# version 作成（各プロジェクト）
npx clasp version "vXXX 変更内容"
npx clasp version "vXXX 変更内容" -P gas/admin
npx clasp version "vXXX 変更内容" -P gas/member

# 固定 deployment を更新（各プロジェクト）
npx clasp redeploy <DEPLOYMENT_ID> --versionNumber <N>
npx clasp redeploy <ADMIN_ID> --versionNumber <N> -P gas/admin
npx clasp redeploy <MEMBER_ID> --versionNumber <N> -P gas/member
⚠️
同期忘れ注意: backend/Code.gs を変更したら必ず gas/admin/Code.gs と gas/member/Code.gs に cp で同期してから push すること（v254教訓）。

セキュリティ

第三者評価に基づく是正タスク（docs/109）— v263 時点で全完了

  • DONE
    processApiRequest deny-by-default
    v261 時点で実装済みを確認・archive。PUBLIC/MEMBER/ADMIN_LOGIN/ADMIN_ACTION_PERMISSIONS の4分類ホワイトリストで完全制御。
  • DONE
    自己操作 API IDOR 修正
    v261 時点で実装済みを確認・archive。loginId からサーバー側で会員ID/職員IDを解決し、クライアント指定値を上書きする。
  • DONE
    パスワードハッシュ PBKDF2-HMAC-SHA256 移行
    v262 で実装。新規アカウントは即時 PBKDF2（10000反復）。既存アカウントはログイン時に自動 rehash（ユーザー操作不要）。
  • DONE
    OAuthスコープ最小化・CIセキュリティゲート
    v263 で実装。全3プロジェクトから不要スコープ削除。npm run security:audit をリリース前必須チェックに追加。
  • DONE
    CM番号編集ポリシー（案C確定）
    v263 で確定。管理者のみ変更可・会員セルフサービスは読み取り専用。docs/05_AUTH_AND_ROLE_SPEC.md §9 に正式記録。

確定済みセキュリティ方針

  • 管理者と会員は完全分離（v250〜確定）。admin URLと member URLは独立した別プロジェクト
  • demo login、mock member route、画面内 demo selector は復活禁止
  • 外部入力は不信入力として扱い、モデル出力をそのまま shell・SQL・HTML・デプロイ設定へ流し込まない
  • 本番 deploy、DB 更新、権限変更は人間承認を前提とする
  • seedDemoData は完全バックアップと明示承認なしでは実行禁止
