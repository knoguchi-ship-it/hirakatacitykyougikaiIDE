⚛️ React 19 + TypeScript パターン

GAS との通信（google.script.run）

React から GAS バックエンドへの通信は、google.script.run を Promise 化した callApi 関数で統一。

// src/shared/api-base.ts
export function callApi<T>(action: string, payload?: unknown): Promise<T> {
  return new Promise((resolve, reject) => {
    google.script.run
      .withSuccessHandler((result: string) => {
        const parsed = JSON.parse(result);
        if (parsed.success) resolve(parsed.data as T);
        else reject(new Error(parsed.error || 'API error'));
      })
      .withFailureHandler((err: Error) => reject(err))
      .processApiRequest(action, JSON.stringify(payload ?? {}));
  });
}

// 使用例
const data = await callApi<Member[]>('fetchAllData');
⚠️ processApiRequest が { success: false, error: '...' } を返した場合、外側の success: true で包まれると callApi が resolve してしまう。返値の data.success も必ず確認すること（v264-fix 教訓）。

function processApiRequest(action, payload) {
  var parsedPayload = parsePayload_(payload) || {};
  var isPublicAction = !!PUBLIC_ALLOWED_ACTIONS[action];
  var isMemberAction = !!MEMBER_ALLOWED_ACTIONS[action];
  var requiredPerms = ADMIN_ACTION_PERMISSIONS[action];

  // deny-by-default: 知らないアクションは全て拒否

    return JSON.stringify({ success: false, error: 'unsupported_action' });

    if (!session || requiredPerms.indexOf(session.adminPermissionLevel) === -1) {
      return JSON.stringify({ success: false, error: 'unauthorized' });
    }
  }
  // 以降、各アクションの処理...
}

🔐 認証設計

PBKDF2-HMAC-SHA256 パスワードハッシュ（v262〜）

旧: 単発 SHA-256 + salt。現行: PBKDF2-HMAC-SHA256（10000反復）で offline cracking 耐性を強化。

// GAS での PBKDF2 実装
function hashPasswordPbkdf2_(password, salt) {
  var bytes = Utilities.computeHmacSha256Signature(password, salt);
  for (var i = 1; i < 10000; i++) {
    bytes = Utilities.computeHmacSha256Signature(bytes, salt);
  }
  return Utilities.base64Encode(bytes);
}

// login 時に旧ハッシュを自動 rehash（ユーザー影響なし）
function verifyPassword_(input, salt, storedHash) {
  var newHash = hashPasswordPbkdf2_(input, salt);
  return { match: newHash === storedHash, needsRehash: isOldFormat(storedHash) };
}

HMAC-SHA256 ステートレストークン（v264-fix〜）

CacheService 非依存。JWTに相当する自己検証型トークン。

// トークン生成
function createPublicIdentityToken_(data, ttlSeconds) {
  var secret = PropertiesService.getScriptProperties()
    .getProperty('PUBLIC_TOKEN_SECRET') || 'fallback_key';
  var payload = JSON.stringify({ d: data, exp: Date.now() + ttlSeconds * 1000 });
  var encoded = Utilities.base64EncodeWebSafe(payload);
  var sigBytes = Utilities.computeHmacSha256Signature(encoded, secret);
  return encoded + '.' + Utilities.base64EncodeWebSafe(sigBytes);
}

// トークン検証
function verifyPublicIdentityToken_(token) {
  var parts = token.split('.');
  if (parts.length !== 2) return null;
  var encoded = parts[0], sig = parts[1];
  var expectedSig = Utilities.base64EncodeWebSafe(
    Utilities.computeHmacSha256Signature(encoded, secret)
  );
  if (sig !== expectedSig) return null; // 改ざん検出
  var obj = JSON.parse(Utilities.newBlob(Utilities.base64DecodeWebSafe(encoded)).getDataAsString());
  if (obj.exp < Date.now()) return null; // 有効期限切れ
  return obj.d;
}
✅ この設計により:（1）サーバー間の共有状態不要（2）スケール可能（3）CacheService 蒸発の影響ゼロ

sendEmailWithValidatedFrom_ パターン（v268〜）

function sendEmailWithValidatedFrom_(to, subject, body, options) {
  // userinfo.email スコープがない場合でも安全に動作
  var ownerEmail = '';
  try { ownerEmail = Session.getEffectiveUser().getEmail(); } catch (e) {}

  var from = String((options && options.from) || ownerEmail).trim();

  if (!from || from === ownerEmail) {
    // from 未指定 or deploying user → MailApp（スコープ要件低い）
    MailApp.sendEmail({ to, subject, body, ... });
    return;
  }
  // エイリアス送信 → GmailApp（管理者 split のみで使用）
  GmailApp.sendEmail(to, subject, body, { from, ... });
}

変更申請セキュリティ原則

  • 既存DB値の非表示: 変更フォームにデフォルト値なし
  • 管理者承認必須: 申請はキューに保存、自動反映なし
  • 本人確認: CM番号+氏名（個人）/ 事業所番号（事業所）
  • 返信メール分離: 連絡先メールはDBと無関係
  • レート制限: 同一ID 15分以内5回まで

GAS 特有の制約まとめ

実行時間・クォータ

  • スクリプト最大実行時間: 6分
  • MailApp.sendEmail: 100通/日（@gmail.com）/ 1500通/日（Workspace）
  • CacheService: 1エントリ最大 100KB、合計 1000エントリ
  • URL Fetch: 1000回/日

Spreadsheet DB の注意点

  • getLastRow() / getLastColumn() は空行を正確に返さない場合がある
  • 大量データは getValues() で一括取得してメモリ内でフィルタリング
  • 書き込みは setValues() でまとめて書く（1セルずつは遅い）
  • LockService で競合を防止（長時間ロック注意）
