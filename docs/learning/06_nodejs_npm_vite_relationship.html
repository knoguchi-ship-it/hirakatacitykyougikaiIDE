Node.js・npm・Vite の関係

この3つは一緒に使われることが多いですが、同じ役割ではありません。 まずは「土台」「命令の窓口」「開発ツール」という3段階で分けて考えると理解しやすくなります。

Node.js

JavaScript をブラウザ外で実行する環境です。

  • 土台になる
  • Vite や TypeScript を動かせる
  • それ自体が必ずサーバーになるわけではない
→

npm

Node.js 上でライブラリ管理やコマンド実行を行う仕組みです。

  • npm install で依存関係を入れる
  • npm run dev で定義済みコマンドを呼ぶ
  • Node.js の上で使う
→

Vite

Node.js 上で動くフロントエンド開発ツールです。

  • 開発サーバーを立てる
  • React / TypeScript をビルドする
  • ブラウザへ配布するコードをまとめる

ブラウザ

Vite が提供するローカルサーバーや、GAS の本番 URL にアクセスして、 実際に画面を表示する場所です。React の画面が見えるのは最終的にブラウザです。

Node.js
実行環境
npm
管理と実行の窓口
Vite
開発サーバーとビルドツール
ブラウザ
画面を実際に見る場所

正確な理解としては、Node.js がローカルサーバーそのものなのではなく、 Node.js の上で Vite が動き、その Vite がローカルサーバーを提供する、です。 そしてブラウザは、そのローカルサーバーにアクセスして画面を表示します。
