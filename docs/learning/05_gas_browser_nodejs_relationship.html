GAS・ブラウザ・Node.js の関係

この案件では、3つの場所で役割が分かれています。 どこで何が動くかを切り分けると、トラブルの原因も追いやすくなります。

Node.js

ローカルで Vite、TypeScript、ビルドスクリプトを動かす。開発と出荷準備の担当。

ブラウザ

React の画面を実際に表示し、ユーザーの操作を受ける。見える部分の担当。

Google Apps Script (GAS)

HTML を返し、API を実行し、Spreadsheet を本番データとして読む/書く。本番バックエンド担当。

開発時

Node.js が Vite を動かす → Vite 開発サーバーがローカル URL を提供する → ブラウザがその URL にアクセスして画面を表示する。

ただしこの案件では、最終的な認証・API・権限・DB 整合の確認は GAS 上で必要です。

本番時

Node.js でビルドした HTML を GAS に置く → ブラウザが GAS の Web アプリ URL にアクセスする → 画面から google.script.run 経由で GAS API を呼ぶ。
