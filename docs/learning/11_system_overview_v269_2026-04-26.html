システム全体概要

枚方市介護支援専門員連絡協議会の会員管理・研修申込・公開申込ポータルを統合した、Google Workspace ベースの会員システムです。

React 19 + TypeScript Google Apps Script Tailwind CSS Spreadsheet DB PBKDF2 認証 HMAC トークン

🏗️システム構成

🌐
公開ポータル
誰でもアクセス可能。研修申込・入会申込・登録情報変更申請・退会申請。認証不要の匿名アクセス。
👤
会員マイページ
会員がログインして利用。プロフィール閲覧・更新、研修申込・取消、年会費確認、パスワード変更。
⚙️
管理者コンソール
Google アカウント認証のホワイトリスト制。会員管理、変更申請承認、研修管理、メール送信、各種設定。

公開ポータルのカード構成

🎓 研修を申し込む
受付中研修の一覧表示・申込・取消（申込ID+登録メール照合）。
📋 新規入会を申し込む
個人・事業所・賛助会員の新規入会申込。認証情報メール自動送信。
✏️ 会員登録情報を変更する
OTPなし本人確認（v264〜）。変更申請→管理者承認→DB反映のワークフロー。既存DB情報は一切表示しない。
🚪 退会を申し込む
同様の本人確認フローで退会申請。管理者承認後に年度末退会処理。

🔒認証・認可

会員認証

  • 方式: ログインID + パスワード
  • ログインID: 個人=CM番号8桁 / 事業所=事業所番号 / 賛助=会員番号
  • パスワードハッシュ: PBKDF2-HMAC-SHA256（10000反復）v262〜
  • rehash: ログイン時に旧SHA-256ハッシュを自動移行
  • アカウントロック: 5回失敗でロック
  • セッション: CacheService（TTL 30分）

管理者認証

  • 方式: Google アカウント + ホワイトリスト照合
  • 照合: Session.getActiveUser().getEmail() → T_管理者Googleホワイトリスト
  • 権限: MASTER / ADMIN / TRAINING_MANAGER / TRAINING_REGISTRAR / GENERAL
  • GIS廃止: v118〜。GAS iframe sandbox との互換性問題のため

公開ポータル 本人確認（v264〜）

OTPなし。HMAC-SHA256署名付きステートレストークンで CacheService 非依存（v264-fix〜）。

管理者ポータル → システム設定 → 「入会・登録メール設定」で全5種を一元管理。
メール種別送信タイミング設定キー
個人・賛助会員向け入会申込時IND_SUPP_EMAIL_ENABLED/SUBJECT/BODY
事業所 代表者向け入会申込時BIZ_REP_EMAIL_ENABLED/SUBJECT/BODY
事業所 メンバー向け入会申込時BIZ_STAFF_EMAIL_ENABLED/SUBJECT/BODY
追加職員向け職員追加申請承認時STAFF_ADD_STAFF_EMAIL_ENABLED/SUBJECT/BODY

テンプレート変数: {{氏名}} {{ログインID}} {{パスワード}} {{会員マイページURL}} {{事業所名}} {{追加職員氏名}}（代表者通知のみ）

📝変更申請ワークフロー（v264〜）

①公開ポータルで申請
→
T_変更申請に PENDING 保存
→
②管理者が内容確認
→
③承認 → DB反映
→
申請者へ通知メール

申請種別

  • MEMBER_UPDATE: 登録情報変更（フィールド変更・職員追加除籍を含む）
  • WITHDRAWAL: 退会申請
  • STAFF_ADD: 職員追加（単独申請）
  • STAFF_REMOVE: 職員除籍（単独申請）

変更可能フィールド（個人）

  • 氏名セット（氏・名・フリガナ氏・フリガナ名）
  • 連絡先（メール・携帯）
  • 勤務先情報（電話・FAX・住所）
  • 自宅住所
  • CM番号 ⚠️（ログインIDも変更）
  • 通知方法・郵送先区分
セキュリティ: 変更内容のデフォルト値は一切表示しない。既存DB情報の漏洩防止。管理者承認なしでDBは変更されない。
