4. 認証アーキテクチャ

会員認証

ログイン ID + パスワード（SHA-256 ハッシュ）で認証。

T_認証アカウント テーブルを照合。Google ログインは不使用。

ID + PW 入力
→
memberLogin_
ハッシュ照合
→
会員マイページ表示

管理者認証（セッション方式）

Session.getActiveUser().getEmail() で Google メールを取得し、T_管理者Googleホワイトリスト と照合。

GAS iframe sandbox では OAuth ポップアップが動作しないため、セッション認証が唯一の安定方式。

ボタン1クリック
→
checkAdminBySession_
メール照合
→
管理コンソール表示

注: IDトークン検証方式（adminGoogleLogin_）は v118 で廃止済み。GIS が GAS iframe sandbox origin で動作しないことが確定したため。

7. パフォーマンス最適化（v150）

キャッシュチャンク分割

CacheService で 100KB 超レスポンスを 90KB/chunk に分割格納。TTL 300 秒。

複合 API

ログイン直後の round-trip を削減する統合 API 3 本（getAdminInitData, memberLoginWithData, adminLoginWithData）。

スケルトン UI

ローディング中に骨格表示（WCAG 2.2 準拠）。体感速度を改善。

8. v151-v154 の主な変更

v152: 年会費一括保存の前年度末退会

ドラフト専用「前年度末退会」を追加。保存時に T_会員 へ前年度末退会処理を反映し、対象年度の年会費レコードを論理削除。

v153: Tailwind CDN 廃止

ビルド済み CSS を単一 HTML へ内包する構成に変更。CDN 依存を排除。

v154: ダッシュボード今年度統一

会員管理ダッシュボードを今年度基準へ統一し、会員一覧の対象年度・会員種別・会員状態・キーワード検索と連動。

v151: キャッシュ二重パースバグ修正

getAdminDashboardData_() のキャッシュ取得で、パース済みオブジェクトを再度 JSON.parse していた問題を修正。
