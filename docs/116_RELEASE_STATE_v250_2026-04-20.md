# Release State: v250 (2026-04-20)

## 概要

管理者 split プロジェクトで `member_unauthorized` エラーが表示されていた根本原因を修正。

## 変更内容

### 症状
- 管理者 split プロジェクト（`gas/admin`）にアクセスし、Google アカウントでログイン成功後、
  メインコンテンツに `member_unauthorized` エラーが赤いボックスで表示されていた。
- サイドバーに「会員マイページ」「研修受講の申込み」が表示されていた（管理者専用画面に不要）。

### 根本原因（2件）

1. **`src/components/Sidebar.tsx`**: `menuItems` 配列に `profile` / `training-apply` が無条件で含まれていた。
   `showMemberPages` prop は定義されていたが、`menuItems` 構築に反映されていなかった。

2. **`src/App.tsx` useEffect (line ~668)**: `currentView === 'profile'` のとき `loadMemberPortalData` を呼ぶ
   ガードに `isAdminShell` チェックがなかった。管理者ログイン直後に React の初期 state
   `currentView='profile'` が一瞬残り、sessionToken なしで会員 API が呼ばれ `member_unauthorized` が返っていた。

### 修正

| ファイル | 変更箇所 | 内容 |
|---|---|---|
| `src/components/Sidebar.tsx` | `menuItems` 構築 | `profile` / `training-apply` を `showMemberPages` フラグで条件化 |
| `src/App.tsx` | `<Sidebar>` 呼び出し | `showMemberPages={!isAdminShell}` を追加 |
| `src/App.tsx` | useEffect (会員データ取得) | `!isAdminShell &&` ガードを追加 |
