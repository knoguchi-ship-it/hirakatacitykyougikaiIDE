## 1. 委託先管理（APPI 第25条）

個人データの取扱いを委託する場合、委託先の適切な監督を行う義務がある。

### 1.1 現行の委託先一覧

| 委託先 | 提供サービス | 委託する個人データの範囲 | 所在地 | 契約形態 |
|--------|------------|----------------------|--------|---------|
| Google LLC | Google Workspace（Spreadsheet, Apps Script, Drive, Gmail） | 会員情報全般（氏名、メールアドレス、所属、連絡先等） | Google のグローバル基盤上で処理。実際の保存場所・リージョン設定は管理コンソール設定値を年次確認する | Google Workspace 利用規約 + データ処理に関する追加条項 (DPAT) |
| Google LLC | Google Cloud Platform（GCP）※移行予定 | 同上 | Google Cloud の設定値に依存 | GCP 利用規約 + データ処理に関する追加条項 |

### 1.2 委託先の選定基準

以下の観点で委託先を選定・評価する:

1. **安全管理措置の水準**: ISO 27001/27017/27018 認証、SOC 2 Type II 報告書の取得状況
2. **契約上の保護**: データ処理契約（DPA）の締結状況
3. **データの所在**: サーバー所在地、データ移転先国の十分性認定の有無
4. **事故発生時の通知体制**: インシデント報告の迅速性に関する契約条項

### 1.3 Google に関する安全管理措置の確認

- **認証**: ISO 27001, ISO 27017, ISO 27018, SOC 2 Type II, FedRAMP
- **データ処理追加条項 (DPAT)**: Google Workspace 管理コンソール > アカウント > 法的情報 > データ処理に関する追加条項 で確認・同意
- **日本との関係**: Google Cloud は APPI に関する情報提供ページを公開している。個人情報保護法上の責務分担は委託元・委託先で異なるため、契約と設定を合わせて確認する
- **データ所在地**: Google Workspace / Google Cloud の実際の保存場所は契約プランと管理コンソール設定に依存する。日本リージョンを利用しているかどうかは、毎回「契約プラン」「データ リージョン設定」「管理者確認日」をセットで記録する
- **本案件の現状**: 2026-03-31 時点では、このリポジトリ内に契約プランとリージョン設定値の確定証跡がないため、「日本リージョン運用済み」とは断定しない

### 1.4 定期確認

- **年1回**: 委託先の認証・契約状況を確認し、本文書を更新する
- **確認担当**: システム管理者（MASTER権限保持者）
- **次回確認予定**: 2027-04-01
- **確認時の必須記録**: 契約プラン、DPAT 同意有無、データリージョン設定値、確認日、確認者、証跡保存場所

## 2. 漏えい等報告手順（APPI 第26条）

2022年4月施行の改正法により、一定の漏えい等が発生した場合、個人情報保護委員会への報告および本人への通知が義務化された。

### 2.1 報告が必要な事態（規則第7条）

以下のいずれかに該当する場合、報告義務が生じる:

1. **要配慮個人情報**の漏えい等（本システムでは該当なし※）
2. **不正アクセス**による漏えい等の**おそれ**
3. 財産的被害のおそれがある漏えい等
4. **1,000人超**の個人データの漏えい等

※ 介護支援専門員の資格情報は要配慮個人情報に該当しない。ただし、将来的に利用者（要介護者）情報を扱う場合は要配慮個人情報に該当する可能性がある。

### 2.2 報告フロー

```
[インシデント検知]
    ↓
[初動対応] 被害拡大防止（アカウント停止、アクセス遮断等）
    ↓
[影響範囲の特定] 漏えい件数・データ種別・原因の特定
    ↓
[速報] 検知から3-5日以内に個人情報保護委員会へ報告
    ↓
[本人通知] 速やかに該当する本人へ通知
    ↓
[確報] 検知から30日以内（不正アクセスの場合60日以内）に確報提出
    ↓
[再発防止策] 原因分析・対策実施・文書化
```

### 2.3 報告先・連絡先

| 項目 | 詳細 |
|------|------|
| 個人情報保護委員会 報告フォーム | 個人情報保護委員会の最新報告フォームを使用する（URL は改訂されうるため報告時に公式サイトで再確認） |
| 報告様式 | PPC所定の様式（上記URLからダウンロード） |
| 電話窓口 | 03-6457-9685（個人情報保護委員会） |
| 内部報告先 | 協議会会長 → システム管理者（MASTER権限保持者） |

### 2.4 速報に記載する事項

1. 漏えい等が発生し、又は発生したおそれがある個人データの項目
2. 漏えい等が発生し、又は発生したおそれがある個人データに係る本人の数
3. 原因
4. 二次被害又はそのおそれの有無及びその内容
5. 本人への対応の実施状況
6. 公表の実施状況
7. 再発防止のための措置
8. その他参考となる事項

### 2.5 本人通知に含める事項

1. 概要（漏えい等の事実）
2. 漏えい等が発生した個人データの項目
3. 原因
4. 二次被害のおそれの内容・対応策
5. 問い合わせ先

### 2.6 Google 側のインシデント通知

Google Workspace の DPAT に基づき、Google はデータインシデントを認識した後速やかに（通常72時間以内）に通知する義務を負う。通知は Google Workspace 管理コンソールのアラートセンターおよび管理者メールに送付される。

## 3. 安全管理措置の概要（APPI 第23条）

### 3.1 組織的安全管理措置

- RBAC 5段階権限モデルによるアクセス制御（MASTER/ADMIN/TRAINING_MANAGER/TRAINING_REGISTRAR/GENERAL）
- 管理者ホワイトリスト（T_管理者Googleホワイトリスト）による管理者認証
- 本手順書による委託先管理・漏えい報告体制の整備

### 3.2 技術的安全管理措置

- パスワードの SHA-256 + salt ハッシュ保存（※将来的に Argon2id/bcrypt への強化を計画）
- Google の暗号化（転送中: TLS、保存時: AES-256）
- Session ベース管理者認証（`Session.getActiveUser()` + メール照合）
- アカウントロックアウト機能

### 3.3 物理的安全管理措置

- データは Google のデータセンターに保存（ISO 27001 認証施設）
- 当協議会はオンプレミスサーバーを保有しない
