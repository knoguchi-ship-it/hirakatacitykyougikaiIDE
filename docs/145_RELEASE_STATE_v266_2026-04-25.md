## 変更内容

### 1. 入会・登録メール設定 UI 統合再設計

旧: 「入会通知メール」（個人・事業所共通 + 完了画面文言）＋「事業所会員メール設定」の2セクションに分散、誰に送るかが不明瞭。

新: 2セクションを明確な責務で分離。

#### 新セクション構成

| セクション | 内容 |
|---|---|
| **入会・登録メール設定** (`settings-all-email`) | 全5種のメール設定を統合。全体マスタースイッチ、送信元、各メール ON/OFF + 件名 + 本文 |
| **入会完了画面の文言設定** (`settings-portal-completion`) | 公開ポータル完了画面の文言のみ。メール設定とは独立 |

#### 入会・登録メール設定の内部構成

```
全体マスタースイッチ（全メール一括停止）
送信元アドレス（全メール共通）

▍入会申し込み時のメール
  個人・賛助会員向け  → ON/OFF + 件名 + 本文 + テンプレート管理
  事業所 代表者向け   → ON/OFF + 件名 + 本文
  事業所 メンバー向け → ON/OFF + 件名 + 本文

▍職員追加申請 承認時のメール
  追加職員向け        → ON/OFF + 件名 + 本文
  代表者通知          → ON/OFF + 件名 + 本文
```

### 2. 個人・賛助会員メール 個別ON/OFF追加（`IND_SUPP_EMAIL_ENABLED`）

旧: 個人・賛助会員は全体フラグのON/OFFのみで制御、個別ON/OFFなし。  
新: 事業所メールと同様に個別 ON/OFF を追加。

| 設定キー | 説明 |
|---|---|
| `IND_SUPP_EMAIL_ENABLED` | 入会時：個人・賛助会員メール ON/OFF |

全体フラグ（`CREDENTIAL_EMAIL_ENABLED`）が最優先の構造は変わらず。

### 3. 優先制御ロジック（確定）

```
CREDENTIAL_EMAIL_ENABLED (全体マスタースイッチ)
  ├── ON → 個別設定が有効
  │   ├── 個人・賛助: IND_SUPP_EMAIL_ENABLED
  │   ├── 事業所代表者: BIZ_REP_EMAIL_ENABLED
  │   ├── 事業所メンバー: BIZ_STAFF_EMAIL_ENABLED
  │   ├── 追加職員: STAFF_ADD_STAFF_EMAIL_ENABLED
  │   └── 代表者通知: STAFF_ADD_REP_EMAIL_ENABLED
  └── OFF → 全メール停止（個別設定に関わらず）
```
