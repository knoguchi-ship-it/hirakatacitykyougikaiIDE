# Release State v231

更新日: 2026-04-17
対象 version: `v231`（GAS version 231）
fixed deployment: member `@231` / public `@231`

## 概要
- 郵便番号入力を 3 桁 + 4 桁の分割 UI に統一しつつ、保存値は従来どおり `123-4567` 形式の単一文字列を維持した。
- 公開入会申込の「送付先事業所名」を、直接入力できないことが直感的に分かる参照専用表示へ変更した。
- 会員マイページの年会費履歴は「当年度・前年度」の連続 2 年を優先表示し、欠損年度は `未納` で補完するよう修正した。

## 変更内容

### 1. 郵便番号 UI の標準化
- `src/components/PostalCodeInput.tsx` を追加。
- 入力 UI は 3 桁欄 + 4 桁欄に分割し、貼り付け時の正規化・自動フォーカス移動・`inputMode="numeric"`・`autocomplete="postal-code"` を統一。
- 親コンポーネントに返す値は `123-4567` 形式の単一文字列のままとし、DB スキーマ変更は行っていない。

### 2. 送付先事業所名の参照専用化
- 公開入会申込フォームで、郵送先が勤務先の場合に表示する「送付先事業所名」を参照専用ボックスへ変更。
- `参照専用` バッジとグレーアウト表示を入れ、編集は上部の「勤務先・所属情報」で行うことを明示。

### 3. 年会費履歴の連続年度表示
- `buildMemberAnnualFeeHistory_()` を修正し、会員マイページでは current fiscal year / previous fiscal year の 2 年を優先表示。
- 当年度または前年度の実レコードが欠損していても、会費対象会員であれば `未納` として補完表示する。
- これにより、2026 年度未納補完時に 2025 年度が 2024 年度へ押し出される表示を防ぐ。
