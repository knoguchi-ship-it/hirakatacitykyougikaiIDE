# リリース記録 v245（2026-04-19）

## 概要

会員マイページ（`MemberForm.tsx`）の事業所職員追加 UI を全面的に見直し、
v244 で残っていたバリデーション不整合を修正した。

## 背景

v244 では `isNew` ドラフト行の概念を導入したが、
`handleSubmit()` 内で `isNew` を除去した後に `validate()` を呼んでいたため、
追加した職員行に対するメールアドレス必須・介護支援専門員番号必須の検証が正しく効かない経路が残っていた。
また、職員追加 UI 自体も入力しづらく、取消導線も弱かった。

## 実施内容

### `src/components/MemberForm.tsx`

1. `handleSubmit()` の処理順を修正し、`validate()` を先に実行するよう変更
2. 新規追加した職員行に `介護支援専門員番号` 入力欄を追加
3. `介護支援専門員番号` を 8 桁半角数字で必須検証
4. 職員行 UI をカード型 2 列グリッドへ再構成
5. `氏名 / フリガナ` を 2 列、`メールアドレス` を全幅で配置
6. `介護支援専門員番号 / 状態・権限` を 2 列で整理
7. `取消` ボタンをカードヘッダー右端へ移動し、見切れにくく修正

### Apps Script / release

1. GAS の 200 バージョン上限に到達していたため、操作者が古いバージョンを手動削除
2. `v245` を新規作成
3. fixed deployment 2 本を `@245` へ同期

## 変更ファイル

- `src/components/MemberForm.tsx`
- `backend/index.html`（build:gas で再生成）
- `backend/index_public.html`（build:gas で再生成）
