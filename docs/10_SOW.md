# SOW（運用・品質保証）

更新日: 2026-03-13
対象: 枚方市介護支援専門員連絡協議会 会員システム

## 1. スコープ
- 会員機能、管理者機能、研修登録・申込機能の継続開発
- GAS本番運用、デプロイ、障害対応、関連ドキュメント維持
- **公開ポータル（非会員研修申込機能）の追加開発**（2026-03-12 スコープ追加）
  - 同一 GAS・同一 DB 上に公開向けポータルを追加
  - 非会員からの研修申込受付（ログイン不要）
  - 研修登録フォームは会員ポータルと統一（共通コンポーネント）
  - 個人情報保護法・医療介護ガイダンスへの対応を含む
- **管理コンソール拡張**（2026-03-13 スコープ追加）
  - 会員管理コンソールと研修管理コンソールの双方向ナビゲーション
  - 同一管理者権限（`T_管理者Googleホワイトリスト` 登録アカウント）で両コンソールにアクセス可能
- **研修メール送信機能**（2026-03-13 スコープ追加）
  - 研修申込者（会員・非会員統合）への一斉・個別メール送信
  - 送信元: スクリプトオーナー の Gmail アドレス or エイリアス（`GmailApp.getAliases()` で選択）
  - Reply-To: ログイン中管理者のメールアドレス（`Session.getActiveUser().getEmail()`）
  - 件名・本文の差し込みタグ（`{{氏名}}` `{{事業所名}}`）対応
  - 共通添付ファイル + Drive 指定フォルダからの個別自動添付（ファイル名=申込者氏名）
  - `GmailApp.sendEmail()` 使用（`gmail.send` スコープを `appsscript.json` に追加）

## 2. 統一仕様（必須）
- 本番URL固定運用を採用する（Deployment ID固定）
- デプロイは `docs/09_DEPLOYMENT_POLICY.md` の手順に従う
- 本番更新は「既存Deployment IDに新Versionを適用」を標準とし、`New deployment` は障害復旧時の例外のみとする
- 本番更新時は以下を必須とする
  - ビルド成功
  - 本番Deployment更新
  - `/exec` 非404確認
  - 変更点のドキュメント反映

## 3. 品質保証基準
- 仕様変更は実装と同時にドキュメント更新
- 404再発時は標準切り分け手順を実施し、結果を記録
- 旧版ドキュメントは `docs/archive` に退避し、現行文書と混在させない
- DB構成の不整合を禁止する（マスタ・テーブル・型定義・実装を同時更新）
- 既存DBに対しては「再作成前提」ではなく「不足定義の追記/移行」で整合を維持する

## 4. 受入条件
- 本番URLが固定されていること
- 最新版のみを本番Deploymentとして維持できる運用であること
- 現行文書のみで運用判断が可能であること
