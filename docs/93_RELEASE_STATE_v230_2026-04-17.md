# Release State v230

更新日: 2026-04-17
対象 version: `v230`（GAS version 230）
fixed deployment: member `@230` / public `@230`

## 概要
- 事業所会員メンバーから個人会員へ変換し、その後もう一度事業所会員メンバーへ戻した際に、旧個人会員が current principal として残る不整合を修正。
- 表示だけではなく、`auth`、`whitelist`、identity 解決の3層で整合を揃えた。

## 変更内容

### 1. current principal 解決
- `checkAdminBySession_` は `T_管理者Googleホワイトリスト.紐付け会員ID` よりも、`紐付け認証ID` が指す `T_認証アカウント.会員ID` を優先する。
- これにより、whitelist 側に古い会員IDが残っていても、管理者セッションは auth の現在主体を参照する。

### 2. 会員種別変換時の追随更新
- `convertStaffToIndividual_` / `convertIndividualToStaff_` に `rebindAuthPrincipal_()` を追加し、変換後の `会員ID` / `職員ID` / `システムロールコード` を auth row に再設定。
- `syncWhitelistMemberLinkByAuthIds_()` により、関連する管理者 whitelist の `紐付け会員ID` も追随更新。

### 3. DB 整合性チェック
- `assertSingleActiveAffiliationByCareManager_()` を追加。
- 同一 `介護支援専門員番号` について、`WITHDRAWN` でない個人/賛助会員と `ENROLLED` の事業所職員が同時に残った場合は変換を失敗させる。

### 4. フロントエンド identity 候補
- `buildLoginIdentities()` で `WITHDRAWN` の個人/賛助会員と `LEFT` の事業所職員を除外。
- 旧主体が会員マイページの選択候補や current identity として残らないよう修正。

## オペレーター確認ポイント
- Google 管理者セッションで、再遷移した本人の会員マイページを開いたとき、旧個人会員ではなく現行の事業所会員メンバーとして表示されること。

- 再遷移後に、退会済みの旧個人会員が identity 候補や表示主体に現れないこと。
