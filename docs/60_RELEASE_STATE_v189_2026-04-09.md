## 実施内容: フロントエンド HTML 圧縮 (Approach B)

### 背景

GAS は HTML レスポンスをサーバー側で gzip 圧縮しない。そのため従来の
`vite-plugin-singlefile` では生 HTML 521 kB がそのままブラウザに送られていた。
`vite-plugin-singlefile-compression` はビルド時に deflate-raw + Base128 圧縮を行い、
ブラウザの `DecompressionStream` API（Chrome 80+）で実行時展開する。

### 変更内容

#### package.json
- `vite-plugin-singlefile` を削除
- `vite-plugin-singlefile-compression@2.2.3` を devDependencies に追加
- `patch-package@8.0.1` を devDependencies に追加
- `postinstall: "patch-package"` を scripts に追加（npm install 後に自動パッチ適用）

#### vite.config.ts
- `import { viteSingleFile } from 'vite-plugin-singlefile'` → `import { singleFileCompression } from 'vite-plugin-singlefile-compression'`
- plugins の `viteSingleFile()` → `singleFileCompression({ compressFormat: 'deflate-raw', useBase128: true })`

#### patches/vite-plugin-singlefile-compression+2.2.3.patch
- プラグインの `config` フック内で `this.meta.rolldownVersion` を参照していたが、
  Vite 6 の `config` フックでは `this` が undefined になるバグ（Rolldown/Vite 7 向けコード）
- `this.meta.rolldownVersion` → `this?.meta?.rolldownVersion` に修正
- `npx patch-package vite-plugin-singlefile-compression` で patches/ に保存済み

### バンドルサイズ計測結果

| 指標 | v188 (before) | v189 (after) | 改善 |
|---|---|---|---|
| member portal (raw HTML) | 522.59 kB | 148.97 kB | -373.62 kB (-71%) |
| member portal (gzip 参考) | 127.75 kB | 129.95 kB | +2.20 kB |
| public portal (raw HTML) | 317.79 kB | 98.48 kB | -219.31 kB (-69%) |
| public portal (gzip 参考) | 84.14 kB | 85.90 kB | +1.76 kB |

**補足**: gzip 参考値は Vite が出力する「もしサーバーが gzip 圧縮した場合」の推定値。
GAS は HTML レスポンスをサーバー側 gzip しないため、実際のダウンロードサイズは raw HTML の値。
従来 522 kB → 149 kB への削減がブラウザの実ダウンロード量削減に直結する。
