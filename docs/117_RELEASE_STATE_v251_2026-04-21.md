# Release State: v251 (2026-04-21)

## 概要

公開ポータルの物理的切り分けが完了。`doGet` を `ScriptApp.getScriptId()` ベースのルーティングに刷新し、URL パラメータに依存しない3プロジェクト完全分離を達成。

---

## 変更内容

### 問題（解消済み）

統合プロジェクトの base URL (`AKfycbxy.../exec`) に `?app=` パラメータなしでアクセスすると、`doGet` のデフォルト値 `'member'` により会員マイページが表示されていた。公開ポータルの URL から会員サイトへの迂回が可能な状態だった。

### 修正: `backend/Code.gs doGet`

`?app=` URL パラメータ依存を廃止し、`ScriptApp.getScriptId()` でプロジェクトを識別して配信ページを固定。

```js
var SCRIPT_ID_ROUTES = {
  '1ZKFJKNr4I...': { file: 'index',        title: '会員マイページ...',    favicon: 'member' },
  '1tlBJ-OJjq...': { file: 'index',        title: '管理者ポータル...',    favicon: 'member' },
  '11YRlyWVgW...': { file: 'index_public', title: '研修・入会申込ポータル...', favicon: 'public' },
};
var route = SCRIPT_ID_ROUTES[ScriptApp.getScriptId()]
  || { file: 'index_public', ... };  // 未知プロジェクトは公開ポータルにフォールバック（安全側）
```

効果:
- 統合プロジェクト → 常に `index_public.html`（公開ポータル）
- 会員 split → 常に `index.html`（会員マイページ）
- 管理者 split → 常に `index.html`（管理者ポータル）
- `?app=member`, `?app=admin` 等の URL パラメータは無効

### 修正: `MEMBER_PORTAL_URL`

```js
// Before (v241〜v250): 統合プロジェクト URL
var MEMBER_PORTAL_URL = 'https://...AKfycbxy.../exec';

// After (v251〜): 会員 split プロジェクト URL
var MEMBER_PORTAL_URL = 'https://script.google.com/macros/s/AKfycbxd_.../exec';
```

影響: 入会申込完了時に送信される認証情報メールの「会員マイページURL」が会員 split URL に変更。
