## 変更内容

### member function body pruning

**目的**: member split の生成済み `Code.gs` から、会員 web 経路から到達しない関数本体を削除し、会員アプリの物理分立を進める。

**修正内容**:
- `scripts/build-member-gas.mjs` にトップレベル関数の到達解析を追加。
- `doGet` / `processApiRequest` から静的に到達できないトップレベル関数宣言を member 生成物から削除。
- member 生成物のトップレベル関数数を 576 から 115 に削減。
- `backend/Code.gs` 正本は維持し、削除は `gas/member/Code.gs` 生成物のみに限定。

## 影響範囲

直接影響:

- `gas/member/Code.gs`
- `scripts/build-member-gas.mjs`

非影響:

- `gas/admin/Code.gs`: v282 `@44` のまま。
- 統合 / 公開 deployment: v280 `@283` のまま。
- DB schema / データ: 変更なし。
- OAuth scope: 変更なし。

残存:

- `checkAdminBySession_` は member 生成物に残存。理由は `updateMember_` が admin / self-service 共用関数であり、静的到達上 `checkAdminBySession_` へ到達するため。
- 次フェーズで member self-service 更新処理を admin 更新処理から分離し、その後 `checkAdminBySession_` と関連 admin helper を member 生成物から削除する。
