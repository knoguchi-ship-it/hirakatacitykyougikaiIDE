## 変更内容

### member self-service core split

目的: member self-service 更新経路を admin 認証 wrapper から分離し、将来の member 生成物から admin 認証処理を安全に外せる構造へ進める。

修正内容:

- `updateMember_` は admin wrapper として維持。
- 新規 `saveMemberCore_` を追加し、実保存処理を core に分離。
- `updateMember_` は必要時のみ `checkAdminBySession_()` を実行し、その後 `saveMemberCore_()` へ委譲。
- `updateMemberSelf_` は `updateMember_` を通さず `saveMemberCore_()` へ直接委譲。
- member self-service 経路から admin 認証 wrapper への直接依存を除去。

## 影響範囲

直接影響:

- `backend/Code.gs`
- `gas/member/Code.gs`
- `gas/admin/Code.gs`

仕様上の影響:

- 会員本人更新の保存処理は従来と同じ core を使う。
- 管理者更新は従来どおり `updateMember_` 経由で admin session を検証する。
- registry / action handler 分離は維持。
- 関数本体 pruning は引き続き停止。
