## 5. テスト実施記録

### 実施日: 2026-03-20（@108 デプロイ済み）

#### テスト中に発見・修正した不具合

**F-05 FAIL → FIX → PASS**
- 症状: ADMIN ロールで操作中にも権限コンボボックスが編集可能だった
- 原因: `MemberForm.tsx` でロールコンボの disabled 判定に `activeStaffRole`（ログイン時のprop）を使用していたが、デモ用操作ユーザー切替は `currentStaff?.role`（内部state）しか更新しないため、ADMIN に切り替えても prop は REPRESENTATIVE のままだった
- 修正: `activeStaffRole` → `currentStaff?.role` に3箇所変更（権限コンボ disabled、className、注記条件、退会セクション表示条件）
- 追加修正: 操作ユーザー選択ラベルに「代表者」を追加（REPRESENTATIVE/ADMIN/STAFF の3段階表示）
- バージョン: v108 で修正デプロイ
