**目的**: v280 の app 別 allowlist を一段進め、member/admin の生成済み `Code.gs` から境界外 action registry を物理的に空にする。

**修正内容**:

  - `PUBLIC_ALLOWED_ACTIONS` を `{}` に置換。
  - `ADMIN_LOGIN_ACTIONS` を `{}` に置換。
  - `ADMIN_ACTION_PERMISSIONS` を `{}` に置換。
- `scripts/build-admin-gas.mjs`
  - `APP_SECURITY_BOUNDARY` を `admin` に固定。
  - `PUBLIC_ALLOWED_ACTIONS` を `{}` に置換。
  - `MEMBER_ALLOWED_ACTIONS` を `{}` に置換。

## 現時点の分離レベル

今回の変更は **Phase 3-a: action registry の物理削減**。

- member split の action registry は member action のみ有効。
- admin split の action registry は admin login / admin permission action のみ有効。
- public / integrated 側は v280 のまま `@283` を維持。
- 関数本体の dead code 削除は未実施。次フェーズで関数単位の依存解析と削除を行う。
