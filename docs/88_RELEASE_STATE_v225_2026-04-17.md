# Release State v225

Updated: 2026-04-17
Version: `225`
Fixed deployments: member `@225` / public `@225`

## Summary

v225 releases three pending change sets together:

- Public membership application notice dialog before submission
- Bulk mail template save/load/delete support with auto-attach default `OFF`
- Public membership application transition rules keyed by `介護支援専門員番号` and `事業所番号`

## Included Changes

### 1. Public Membership Notice Dialog

- Replaced the always-visible notice block with a dialog opened from the membership application screen.
- Removed the `IMPORTANT` badge and the full-text body from the main screen.
- Added direct links to the association membership/withdrawal page and incorporation page inside the dialog.

### 2. Bulk Mail Template Management

- Added bulk-mail template save/load/delete functions backed by `T_システム設定.BULK_MAIL_TEMPLATES`.
- Added UI controls in the bulk-mail console to reuse saved subject/body combinations.
- Changed Drive auto-attachment to default `OFF` while preserving the existing send path when enabled.

### 3. Membership Application Transition Rules

- Public `submitMemberApplication_` now resolves existing registrations by `介護支援専門員番号`.
- Individual application:
  - If the applicant is an active business staff member, convert to individual member.
  - If the applicant is already an active individual member, reject as duplicate.
- Business application:
  - Reject duplicate office registration by active `事業所番号`.
  - If a submitted staff member is an active individual/support member, withdraw that member and register as staff of the new office.
  - If a submitted staff member belongs to another active office, withdraw from the current office and transfer to the new office.
- When the transferred source staff is `REPRESENTATIVE`, the backend auto-selects a successor from remaining enrolled staff, or withdraws the source office when no enrolled staff remain.
- Success UI now surfaces transition outcomes to the applicant.
