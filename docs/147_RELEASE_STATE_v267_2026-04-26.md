## 変更内容

### 公開ポータル ヒーローセクション FOIC 修正

**問題**: ポータルトップのヒーロータイトル（見出し）が、設定ロード完了前にデフォルト文字列（"研修申込・申込取消・新規入会申込を受け付けています"）を瞬間表示していた。

**原因**: `renderHome()` が `portalContentSettings ?? DEFAULT_PUBLIC_PORTAL_CONTENT_SETTINGS` でフォールバックしており、API レスポンス到着前にデフォルト値を即座にレンダリングしていた。カードセクションは既に `trainingMenuEnabled === null` チェックでスケルトン制御済みだったが、ヒーローセクションには同等の制御がなかった。

**修正内容**:
- `portalContentSettings === null`（ロード中）の間、ヒーローセクションをデフォルト文字ではなく**構造マッチング型スケルトン**（animate-pulse）で表示
- 設定が確定した後のみ実際の文言をレンダリング
- デフォルト文字列はユーザーに一切表示されない

**適用ベストプラクティス**: 2026年現在の React コンテンツローディング標準。コンテンツ寸法に合わせた構造マッチングスケルトンにより、レイアウトシフトを防止。

**変更ファイル**: `src/public-portal/App.tsx`
