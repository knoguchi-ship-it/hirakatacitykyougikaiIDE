## 緊急対応

### member function pruning の撤回

**発生事象**:

- v283 member split で会員マイページ実行時に `ReferenceError: addPublicStaffMember_ is not defined` が発生。
- 原因は、生成時の到達不能関数削除で `addPublicStaffMember_` 本体を削除した一方、後段の wrapper 代入 `var _origAddPublicStaffMember = addPublicStaffMember_;` が残ったこと。

**対応**:

- member fixed deployment を一旦 v282 相当の `@34` へ切り戻し。
- `scripts/build-member-gas.mjs` の関数本体 pruning 実行を無効化。
- member 生成物を再生成し、`addPublicStaffMember_` 本体と wrapper が両方存在することを確認。
- member split version `36` を作成し、fixed deployment を `@36` へ同期。

## 再発防止

- 単純な静的到達解析による GAS 関数本体 pruning は禁止。
- 関数本体削除を再開する場合は、少なくとも次を検出対象に含める:
  - `var x = functionName_;`
  - `functionName_ = function(...) { ... }`
  - wrapper / monkey patch 形式の再代入
  - Apps Script エディタから直接実行する運用関数
  - trigger / time-driven function
- 次フェーズは関数削除ではなく、admin 更新処理と member self-service 更新処理の分離設計に戻す。
