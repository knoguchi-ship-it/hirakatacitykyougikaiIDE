# Release State: v204

Date: 2026-04-14
Version: `204`
Fixed deployments: member `@204` / public `@204`

## Summary

名簿PDF出力の50件上限を廃止。1回のボタン押下で全件を一括出力できるようにした。
`UrlFetchApp.fetchAll()` による並列 PDF 生成で GAS 6分制限内に全件処理を完了する。

## Changes

### GAS backend (`backend/Code.gs`)

- `generateRosterZip_` を並列バッチ処理へ全面改修
  - 旧: 1本の temp SS で1件ずつ順次処理（~4-6s/件 × 50件 → 約3〜5分でも50件まで）
  - 新: `PARALLEL_BATCH=10` 本の temp SS を使い回し、`UrlFetchApp.fetchAll()` で並列 PDF 取得
  - 旧の `ROSTER_PDF_MAX_BATCH = 50` 上限チェック（throw）を廃止
  - 会員種別ごとに memberIds を事前ソートし、各 temp SS の `showSheet/hideSheet` 呼び出し回数を最小化
    - 同じ会員種別が続く間は `lastType` を追跡してシート表示切り替えをスキップ
  - バッチループ: `PARALLEL_BATCH` 件ずつ → write → `SpreadsheetApp.flush()` 1回 → `getValue()` → `fetchAll()` → blob 収集
  - 終端で全 temp SS を `setTrashed(true)` でクリーンアップ
  - 推定処理時間（203件）: ~8s/バッチ × ~14バッチ ≈ 2〜3分（GAS 6分制限内）
  - `UrlFetchApp.fetchAll()` は `muteHttpExceptions: true` で個別エラーを収集し、失敗分はエラー一覧として返す

### Frontend (`src/components/RosterExport.tsx`)

- `const ROSTER_MAX_BATCH = 50` を削除
- `const overLimit = effectiveTargets.length > ROSTER_MAX_BATCH` を削除
- 対象件数ヘッダーの overLimit 警告バッジを削除
- ZIP生成ボタンの `disabled` 条件から `overLimit` を除去
- フッターの「50件以下に絞ってください。」エラー文を削除
- 生成スピナーの説明文を「並列処理中・数分かかる場合あり」に更新
