## 変更内容

### explicit admin-only member core options

目的: 分割前の機能を変えず、member self-service と admin 更新の責務境界をさらに明確化する。

修正内容:

- `saveMemberCore_` に `enableAdminRoleValidation` / `enableAdminAudit` option を追加。
- `updateMember_` は admin wrapper として両 option を `true` にして `saveMemberCore_` を呼ぶ。
- `updateMemberSelf_` は両 option を渡さないため、既定値 `false` で会員セルフ更新を保存する。
- 管理者更新では従来どおり代表者ロール遷移検証と管理者監査ログを実行する。
- 会員セルフ更新では admin-only の代表者検証・管理者監査ログを実行しない。
- DB スキーマ、画面導線、会員ログイン仕様、管理者ログイン仕様は変更なし。

---

## 設計基準

本変更は 2026-04-28 時点の一次情報を確認し、以下の基準に合わせた。

- OWASP ASVS 5.0: access control / authorization の検証基準に沿い、境界外 action は deny by default とする。
- OWASP Authorization Cheat Sheet: least privilege と every request authorization の考え方に沿い、app 境界と wrapper 責務を明示する。
- CISA Secure by Design: 安全な既定値と不要権限の削減を優先する。
- Google Apps Script Deployments: immutable version と fixed deployment の同期を標準運用とする。
