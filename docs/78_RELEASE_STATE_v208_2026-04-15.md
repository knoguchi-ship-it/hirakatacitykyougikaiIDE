## Bugs Fixed

### Bug 1: 事業所会員の名前が姓名になっていた

**原因**: `generateMailingListExcel_()` で全会員に `姓 + 名` を使って `displayName` を計算していた。
事業所会員（BUSINESS）の場合、T_会員 に格納されている `姓名` はシステム的な値であり、
宛名として使うべきは `勤務先名`（事業所名）。

**修正**: `mtype === 'BUSINESS'` の場合は `m['勤務先名']` を `displayName` として使用。

```javascript
// 修正前
var lastName    = String(m['姓']  || '').trim();
var firstName   = String(m['名']  || '').trim();
var displayName = (lastName + ' ' + firstName).trim();

// 修正後
var displayName;
if (mtype === 'BUSINESS') {
  displayName = String(m['勤務先名'] || '').trim();
} else {
  var lastName  = String(m['姓'] || '').trim();
  var firstName = String(m['名'] || '').trim();
  displayName   = (lastName + ' ' + firstName).trim();
}
```

### Bug 2: 住所不備シートの内容が xlsx に反映されなかった

**原因**: GAS の `setValues()` は書き込みをバッファリングする。
`UrlFetchApp.fetch()` で xlsx エクスポートする際に HTTP リクエストが送られるが、
この時点でバッファがまだフラッシュされていない場合、サーバー側の古い状態でエクスポートされる。

データ量が多いシート（事業所会員・個人会員・賛助会員）は大量 `setValues()` の副作用で
偶然フラッシュされることがあるが、行数が少ない住所不備シート（1〜数件）は
フラッシュされないまま HTTP リクエストが走るため、出力されない。

カウント（`rowsInvalid.length`）は JavaScript メモリ上の値なので常に正確。
見た目は「件数は1、シートには0行」という矛盾した状態になる。

**修正**: 4シートへの書き込み完了後、`UrlFetchApp.fetch()` の直前に
`SpreadsheetApp.flush()` を呼び出して強制フラッシュ。

```javascript
// 全シート書き込み完了後
SpreadsheetApp.flush();  // ← 追加

// xlsx エクスポート
var response = UrlFetchApp.fetch(exportUrl, { ... });
```
