## このリリースで対応した課題

### 1. セイ・メイ入力 バリデーション改善（v220 / GAS 220）

**対象ファイル**: `src/components/MemberDetailAdmin.tsx`, `src/components/MemberForm.tsx`

#### 変更前の問題
- 入力中に半角カナ以外の文字を打つと即座に除去されていた（`normalizeKanaInput` が onChange で動作）
- 全角カナ・ひらがなを入力すると文字が消えるため、IME 変換中に意図しない挙動が起きた

#### 変更後の仕様
- **入力中**: 全文字を許可（`normalizeKanaInput` 廃止）
- **保存時**: 自動的に全角カナ・ひらがな → 半角カナに変換（`toHalfWidthKana` 関数）してから検証
- 変換は濁音・半濁音を含む完全対応（ガ→ｶﾞ、パ→ﾊﾟ 等）

#### 実装詳細
- `toHalfWidthKana(value)` をファイルスコープ定数として追加（両ファイルに同一実装）
  - ひらがな → 全角カナ（U+3041–U+3096 を +0x60 シフト）
  - 全角カナ → 半角カナ（`fullToHalf` 対応表）
- `MemberDetailAdmin.tsx`:
  - `validateAllRequired()` に `overrideForm?: Partial<EditableMemberForm>` パラメータ追加
  - `handleSave()` で `convertedForm`（kana 変換済み）を生成し `validateAllRequired(convertedForm)` に渡す
  - `staff` の `lastKana` / `firstKana` も同時変換
- `MemberForm.tsx`:
  - `validate()` に `memberOverride?: Member` パラメータ追加
  - `handleSubmit()` で `kanaConvertedMember` を生成し `validate(kanaConvertedMember)` に渡す
  - `normalizedMemberBase` / `normalizedMember` も `kanaConvertedMember` ベースに統一

### 2. 管理コンソール 年度内在籍判定 修正（v220 / GAS 220）

**対象ファイル**: `backend/Code.gs`（`getAdminDashboardData_` 内）

#### 変更前の問題
- 退会済みメンバーは `memberStatus === 'WITHDRAWN'` で一律カウント外
- 2026-03-31 退会 → 2025年度（4/1/2025〜3/31/2026）も退会済み扱いとなっていた

#### 変更後の仕様
- **在籍判定式**: `入会日 ≤ 年度末 AND（退会日なし OR 退会日 ≥ 年度始）`
- 例: 退会日 2026-03-31 → 2025年度は在籍（3/31 ≥ 4/1/2025）、2026年度は退会済（3/31 < 4/1/2026）

#### 実装注意
v220 の初期実装では `new Date(rawDate + 'T00:00:00')` を使用したが、GAS の `getValues()` が
日付セルを `Date.toString()` 形式（"Mon Apr 01 2019..."）で返す場合に Invalid Date になるバグがあった。
v221 でこのバグを修正（下記参照）。

### 3. 管理コンソール カード数値 joinedDate 空会員の除外バグ修正（v221 / GAS 221）

**対象ファイル**: `src/App.tsx`（`getMemberStatusAtFiscalYear` / `getStaffStatusAtFiscalYear`）、`backend/Code.gs`

#### 変更前の問題

**フロントエンド**（主因）:
```typescript
const joined = parseDateString(member.joinedDate);
if (!joined) return null;  // ← joinedDate が空なら全年度から除外
```
`joinedDate` が未設定の会員が全年度フィルターから除外されていた。
宛名リストは `status=ACTIVE` のみ見るため、この 9 名が宛名リスト(161)と管理コンソール(152)の
差異の原因となっていた。

**バックエンド**（v220 の不具合修正）:
```javascript
// 修正前
var jdObj = joinedDateRaw ? new Date(joinedDateRaw + 'T00:00:00') : null;
// → GAS Date.toString() 形式の文字列に +T00:00:00 を付けると Invalid Date
```

#### 変更後

**フロントエンド** (`App.tsx`):
```typescript
// getMemberStatusAtFiscalYear / getStaffStatusAtFiscalYear 両方
const joined = parseDateString(member.joinedDate);
// joinedDate なし = データ不備として在籍扱い（宛名リストと整合）
if (joined && joined > end) return null;  // 年度終了後に入会した場合のみ除外
```

**バックエンド** (`Code.gs`):
```javascript
var jdNorm = normalizeDateInput_(joinedDateRaw);          // YYYY-MM-DD に正規化
var jdObj = jdNorm ? new Date(jdNorm + 'T00:00:00+09:00') : null;
var wdNorm = (withdrawnDateRaw && memberStatus === 'WITHDRAWN')
  ? normalizeDateInput_(withdrawnDateRaw) : '';
var wdObj = wdNorm ? new Date(wdNorm + 'T00:00:00+09:00') : null;
```
`normalizeDateInput_` 経由で正規化し `+09:00` 付きでパース。`String(dateObj)` 形式への依存を排除。

#### 数値の期待値（修正後）
- 管理コンソール 2026年度 個人会員 ≒ 宛名リスト（広報誌発送）個人会員
- 2026年度内に退会した会員が増えるほど管理コンソール ≥ 宛名リスト になる（正常動作）

## 落とし穴・ノウハウ

### GAS での日付比較に関する注意
- `getValues()` は日付セルを JavaScript `Date` オブジェクトとして返す
- `String(dateObj)` = `"Mon Apr 01 2019 00:00:00 GMT+0900 (Japan Standard Time)"` 形式
- この文字列に `'T00:00:00'` を付加すると `new Date()` が Invalid Date を返す
- 正しい手順: `normalizeDateInput_(raw)` で `"YYYY-MM-DD"` に変換 → `new Date(normalized + 'T00:00:00+09:00')`
- 参照実装: `shouldAutoDeleteOnNextApril_()` (Code.gs)

### フロントエンドの在籍判定カード数値
- ダッシュボードのカード数値（対象会員数・個人会員・事業所会員）は **フロントエンドで再計算**
- `adminDashboardData.memberCount` / `individualCount` 等のバックエンド値はカードに**使用されない**
- フロントエンドは `adminDashboardData.memberRows`（全会員リスト）を受け取り、
  `getMemberStatusAtFiscalYear` で年度フィルターをかけた `filteredAdminMemberRows` からカード値を計算
- バックエンドのカード数値はキャッシュ目的で返しているが、現状 UI では参照されていない

### 宛名リストと管理コンソールの数値が完全一致しない理由（仕様）
| 項目 | 管理コンソール | 宛名リスト |
|------|--------------|-----------|
| 個人会員の定義 | INDIVIDUAL + SUPPORT | INDIVIDUAL のみ（SUPPORT は賛助会員列） |
| 在籍判定 | 年度内在籍（入会日・退会日考慮） | 現在 ACTIVE/WITHDRAWAL_SCHEDULED |
| 年度内退会者 | 計上される（退会済として） | 計上されない |
