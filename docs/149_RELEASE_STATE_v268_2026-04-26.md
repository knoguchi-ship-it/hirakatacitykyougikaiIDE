## 変更内容

### メール送信エラー修正: `Session.getEffectiveUser` スコープ不足

**エラー内容**:
```
sendCredentialEmail_ failed for k.noguchi@hcm-n.org:
指定された権限では Session.getEffectiveUser を呼び出すことができません。
必要な権限: https://www.googleapis.com/auth/userinfo.email
```

**根本原因**: v263 の OAuth スコープ最小化で `userinfo.email` を統合・会員 split から削除したが、メール送信関数 `sendEmailWithValidatedFrom_()` と `sendCredentialEmail_()` が `Session.getEffectiveUser().getEmail()` を無条件で呼び出していた。

| プロジェクト | `userinfo.email` スコープ | 結果 |
|---|---|---|
| 統合（公開） | なし（v263削除） | **エラー** |
| 会員 split | なし（v263削除） | **エラー** |
| 管理者 split | あり | 正常 |

**修正内容**:

1. `sendEmailWithValidatedFrom_()`: `Session.getEffectiveUser()` を try-catch で安全に取得。取得失敗時は空文字にフォールバック。`from` 未指定の場合は `MailApp.sendEmail()`（`userinfo.email` スコープ不要）を使用。

2. `sendCredentialEmail_()`: `replyTo` の fallback を `Session.getEffectiveUser()` から `''` に変更（`sendEmailWithValidatedFrom_()` 側で処理されるため不要）。

**影響範囲**:
- 統合プロジェクト（公開ポータル）: 入会申込メールが送れなかった問題を解消
- 会員 split: 同様の潜在問題を解消
- 管理者 split: 変更なし（`userinfo.email` スコープ保持済み）

**変更ファイル**: `backend/Code.gs`, `gas/admin/Code.gs`, `gas/member/Code.gs`
