## 変更内容

### app 境界 API gate の導入

**目的**: `public / member / admin` の3境界を、UIだけでなく `processApiRequest` の入口でも分離する。

**修正内容**:
- `APP_SECURITY_BOUNDARY` を各 `Code.gs` に追加。
  - `backend/Code.gs`: `public`
  - `gas/member/Code.gs`: `member`
  - `gas/admin/Code.gs`: `admin`
- `processApiRequest` に `isActionAllowedForCurrentApp_()` を追加し、app 境界外の action を `unsupported_action` で拒否する。
- 会員 shell のログイン文言から管理者ログイン説明を除外した。

---

## 現時点の分離レベル

今回の変更は **Phase 1: 到達性の分離** であり、関数本体の完全削除ではない。

- member app から admin action は API 入口で拒否される。
- admin app から member action は API 入口で拒否される。
- public app から member/admin action は API 入口で拒否される。
- 既存機能破損を避けるため、未使用関数の削除・ファイル分割・scope 最小化は次フェーズへ送る。

---

## 完全分立への次フェーズ計画

### Phase 2: app 別 action registry の物理分離

- `PUBLIC_ALLOWED_ACTIONS` / `MEMBER_ALLOWED_ACTIONS` / `ADMIN_ACTION_PERMISSIONS` を app 別ファイルまたは生成単位へ分ける。
- member artifact から admin action 定義を削除する。
- admin artifact から member action 定義を削除する。
- public artifact から member/admin action 定義を削除する。

### Phase 3: Code.gs の dead code 削除

- member `Code.gs` から管理者ログイン、管理者ダッシュボード、管理設定、全件取得、管理者メール/Drive 管理系処理を削除する。
- admin `Code.gs` から member self-service login/write 専用処理を削除する。ただし管理者が会員データを編集する管理系処理は残す。
- public `Code.gs` から member/admin bootstrap と内部 action を削除する。

### Phase 4: manifest / scope 最小化

- public: anonymous public portal に必要な最小 scope のみ。
- member: 会員セルフサービスに必要な Spreadsheet/Mail 等の最小 scope のみ。管理者用 Gmail settings / Drive 管理 scope は持たない。
- admin: 管理者運用に必要な scope を保持し、理由を `docs/09_DEPLOYMENT_POLICY.md` に記録する。

### Phase 5: shared 生成設計

- 共通 DB mapper / date utility / training availability / schema helper を shared 化する。
- shared から app 固有 API へ逆依存しない。
- build 時に app 別 `Code.gs` を生成する方式へ移行する。
