MCP（Model Context Protocol）とは

Model Context Protocol（MCP） は Anthropic が策定したオープン標準で、 AI エージェントと外部ツール・データソースを接続するためのプロトコルです。 MCP サーバーを接続することで、AI がファイルシステム操作・Git 操作・メモリ管理などを 直接実行できるようになります。

🧠

memory

会話をまたいだ情報の永続化。プロジェクトのコンテキストや決定事項を記憶させておける

🧙

sequential-thinking

複雑な問題を段階的に分解・構造化する思考支援。多ステップタスクの品質向上

📂

filesystem

ファイルシステムへの安全なアクセス。指定パス（C:/VSCode）配下を読み書き可能

📦

git

Git リポジトリ操作（ステータス確認・差分取得・コミット・ブランチ管理等）

🎭

playwright

ブラウザ自動操作（ナビゲート・クリック・スクリーンショット・フォーム入力等）

ℹ️

MCP サーバーは stdio（ローカルプロセス） 方式で動作します。 AI がツールを呼び出すたびに npx 経由でサーバープロセスが起動し、 標準入出力でやり取りを行います。

CLI コマンドによる正式な登録方法（推奨）

公式ドキュメントでは、claude mcp add --scope user コマンドが ユーザースコープ MCP の正式な登録方法として推奨されています。 このコマンドは設定を ~/.claude.json に書き込みます。

⚠️

Windows (非 WSL) 環境では npx を直接 command に指定すると 「Connection closed」エラーになる場合があります。 公式ドキュメントでは cmd /c ラッパーの使用が推奨されています。 詳細は セクション 7（Windows 注意事項） を参照してください。

bash — CLI 登録コマンド（user スコープ） 
# memory
$ claude mcp add --scope user --transport stdio memory -- cmd /c npx -y @modelcontextprotocol/server-memory

# sequential-thinking
$ claude mcp add --scope user --transport stdio sequential-thinking -- cmd /c npx -y @modelcontextprotocol/server-sequential-thinking

# filesystem（C:/VSCode 配下を対象）
$ claude mcp add --scope user --transport stdio filesystem -- cmd /c npx -y @modelcontextprotocol/server-filesystem C:/VSCode

# git
$ claude mcp add --scope user --transport stdio git -- cmd /c npx -y @cyanheads/git-mcp-server

# playwright（既存 / 同様の方法で登録する場合）
$ claude mcp add --scope user --transport stdio playwright -- cmd /c npx -y @playwright/mcp@0.0.68

MCP スコープの種類

スコープ フラグ 保存先 適用範囲
user --scope user ~/.claude.json 全プロジェクト（個人専用）
local --scope local（省略可） ~/.claude.json のプロジェクトパス以下 現在のプロジェクトのみ（個人専用）
project --scope project .mcp.json（プロジェクトルート） 現在のプロジェクトのみ（チーム共有可）

管理コマンド一覧

bash 
# 登録済み MCP サーバー一覧
$ claude mcp list

# 特定サーバーの詳細確認
$ claude mcp get memory

# サーバー削除
$ claude mcp remove memory

# セッション内での MCP 状態確認

TOML 設定の全オプション（参考）

TOML — 全オプション例 
[mcp_servers.example-server]
command = "npx"                     # 起動コマンド（必須）
args    = ["-y", "some-mcp-server"]  # コマンド引数
enabled = true                      # false にすると削除せず無効化
startup_timeout_sec = 10            # 起動タイムアウト（デフォルト 10秒）
tool_timeout_sec    = 60            # ツール実行タイムアウト（デフォルト 60秒）
required = false                    # true: 起動失敗時に Codex も起動失敗
🚨

既知の不具合： Codex CLI の特定バージョンで config.toml の MCP サーバーが認識されないケースが報告されています。 セクション名を mcp_servers（アンダースコア） にすることが重要です。 mcp-servers（ハイフン）は動作しません。

信頼済みプロジェクト内に .codex/config.toml を置くとプロジェクト限定になります。

MCP サーバー詳細

🧠
memory ✓ 設定済み
@modelcontextprotocol/server-memory

用途: 会話をまたいだ情報の永続化（エンティティグラフ形式のメモリ）

できること:

  • プロジェクト固有の決定事項・コンテキストをセッション間で記憶
  • 「前回話したこと」を引き継いで連続した作業が可能
  • 人物・タスク・システム情報などをエンティティとして管理

使い方例: 「このプロジェクトのDB接続情報を覚えておいて」「前回の続きから始めて」

🧙
sequential-thinking ✓ 設定済み
@modelcontextprotocol/server-sequential-thinking

用途: 複雑な問題を段階的に分解・構造化する思考支援ツール

できること:

  • 多ステップの実装タスクを整理・順序立て
  • 「思考の途中で考えが変わった」場合に前のステップを修正
  • 仮説立案 → 検証 → 結論の構造的思考

使い方例: 「この設計の問題点を段階的に分析して」「リリース手順を構造化してまとめて」

📂
filesystem ✓ 設定済み
@modelcontextprotocol/server-filesystem

対象ディレクトリ: C:/VSCode（配下の全プロジェクトが対象）

できること:

  • ファイル読み書き・ディレクトリ一覧・ファイル検索
  • ファイル移動・コピー・削除
  • 指定した許可パス配下のみ操作可能（セキュリティ境界あり）

セキュリティ注意: C:/VSCode 配下のファイルを MCP 経由で読み書きできます。機密ファイルの配置には注意してください。

📦
git ✓ 設定済み
@cyanheads/git-mcp-server

用途: Git リポジトリ操作

できること:

  • git status / diff / log の取得
  • ブランチ一覧・切り替え・作成
  • コミット・ステージング操作
  • ファイルの git blame 参照

使い方例: 「今の変更差分を確認して」「main との差分を見せて」

🎭
playwright ✓ Connected 既存・稼働中
@playwright/mcp@0.0.68

用途: Chromium ブラウザの自動操作

できること:

  • URL ナビゲーション・クリック・フォーム入力・キー操作
  • スクリーンショット・DOM スナップショット取得
  • ネットワークリクエスト監視・コンソールログ取得
  • ファイルアップロード・ダイアログ操作

Windows 環境での注意事項

🚨

重要： Windows ネイティブ環境（非 WSL）では、 npx を command に直接指定すると 「Connection closed」エラーになることがあります。 公式ドキュメントでは cmd /c ラッパーの使用が推奨されています。

問題の原因

Windows は Unix シェルのように npx を直接実行できないため、 cmd.exe 経由で実行する必要があります。

CLI コマンド登録時（推奨方法）

bash — Windows 向け cmd /c ラッパー形式 
# NG: Windows では npx を直接実行できない場合がある
$ claude mcp add --scope user memory -- npx -y @modelcontextprotocol/server-memory

# OK: cmd /c 経由で npx を実行
$ claude mcp add --scope user memory -- cmd /c npx -y @modelcontextprotocol/server-memory

settings.json に直接書く場合

JSON — Windows 対応版（cmd /c ラッパー） 
{
  "mcpServers": {
    "memory": {
      "type": "stdio",
      "command": "cmd",
      "args": ["/c", "npx", "-y", "@modelcontextprotocol/server-memory"]
    }
  }
}

現在の settings.json は command: "npx" 形式で設定されています。 接続エラーが出た場合は、上記の cmd /c 形式に変更してください。

トラブルシューティング

🔴 Connection closed エラー（Claude Code / Windows）

原因: Windows ネイティブ環境で npx を直接 command に指定している

対処: command: "cmd", args: ["/c", "npx", ...] 形式に変更する

詳細は セクション 8（Windows 注意事項） を参照

🔴 Codex で MCP が認識されない

確認事項 1: セクション名が [mcp_servers.xxx]（アンダースコア）になっているか確認。mcp-servers（ハイフン）は無効。

確認事項 2: Codex を再起動しているか確認。

確認事項 3: 特定バージョンでの既知バグ。codex --version で確認し、最新版に更新する。

bash — 一時的に無効化する方法
# 問題のサーバーを一時無効化
[mcp_servers.problematic-server]
enabled = false

🔴 settings.json の mcpServers が無視される（Claude Code）

原因: Claude Code はユーザースコープの MCP を ~/.claude.json に保存するため、settings.json の mcpServers キーが一部バージョンで無視される既知の問題があります。

対処: claude mcp add --scope user コマンドで正式登録することを推奨します。

bash — CLI による user スコープ登録 
$ claude mcp add --scope user --transport stdio memory -- cmd /c npx -y @modelcontextprotocol/server-memory
$ claude mcp add --scope user --transport stdio sequential-thinking -- cmd /c npx -y @modelcontextprotocol/server-sequential-thinking
$ claude mcp add --scope user --transport stdio filesystem -- cmd /c npx -y @modelcontextprotocol/server-filesystem C:/VSCode
$ claude mcp add --scope user --transport stdio git -- cmd /c npx -y @cyanheads/git-mcp-server

🔴 npx の初回起動が遅い / タイムアウトする

原因: 初回実行時にパッケージをダウンロードするため時間がかかる

対処 1: 事前にグローバルインストールする

bash — グローバルインストール 
npm install -g @modelcontextprotocol/server-memory
npm install -g @modelcontextprotocol/server-sequential-thinking
npm install -g @modelcontextprotocol/server-filesystem
npm install -g @cyanheads/git-mcp-server

対処 2: Codex の startup_timeout_sec を増やす

TOML
[mcp_servers.memory]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-memory"]
startup_timeout_sec = 30  # デフォルト 10秒 → 30秒に増加

🔴 VS Code で MCP サーバーが表示されない

確認事項 1: mcp.json のルートキーが "servers"（"mcpServers" ではない）になっているか確認

確認事項 2: Ctrl+Shift+P → Developer: Reload Window でリロード

確認事項 3: Ctrl+Shift+P → MCP: Open User Configuration でファイルパスを確認

🔴 Claude Code でグローバル設定の MCP が一部だけ動く（2026-04-17 実例）

⚠️

重要：プロジェクトレベル設定はグローバル設定を完全に上書きする
Claude Code は ~/.claude.json の projects[プロジェクトパス].mcpServers が存在すると、 グローバルの ~/.claude/settings.json の mcpServers を無視します。

症状:

  • グローバルに5つの MCP を設定しているのに、playwright だけが動作する
  • /mcp や ToolSearch で memory / filesystem / git / sequential-thinking が見つからない
  • 各パッケージは手動実行すると正常起動する

根本原因:

SessionStart フック（ensure-playwright.js）が ~/.claude.json のプロジェクトエントリに playwright だけを書き込んでいた。 プロジェクトレベルの mcpServers が存在するとグローバル設定が完全に無効化されるため、 playwright 以外の4つが見えなくなった。

診断手順:

bash — プロジェクトレベルの MCP 設定を確認
# ~/.claude.json のプロジェクトエントリを確認
# 該当プロジェクトのキーに mcpServers があるか確認する
$ node -e "const d=require('/c/Users/kenno/.claude.json'); const p=d.projects['C:/VSCode/YOUR_PROJECT']; console.log(JSON.stringify(p?.mcpServers,null,2))"

対処: ~/.claude.json の該当プロジェクトの mcpServers に、グローバルと同じ全サーバーを追記する。

JSON — ~/.claude.json プロジェクトエントリへの全 MCP 追加例 
// ~/.claude.json → projects["C:/VSCode/YOUR_PROJECT"] → mcpServers
{
  "playwright": { "type": "stdio", "command": "npx", "args": ["-y", "@playwright/mcp@0.0.68"], "env": {} },
  "memory": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-memory"] },
  "sequential-thinking": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"] },
  "filesystem": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "C:/VSCode"] },
  "git": { "type": "stdio", "command": "npx", "args": ["-y", "@cyanheads/git-mcp-server"] }
}

再発防止: SessionStart フック等でプロジェクトレベルに MCP を書き込む場合は、 既存の全エントリを保持したまま追記するロジックを実装すること。 単純な上書き・新規書き込みはグローバル設定を隠してしまう。
