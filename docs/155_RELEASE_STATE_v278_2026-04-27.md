## 変更内容

### 研修表示・申込判定モデルの整理

**問題**: `T_研修.研修状態コード` が「研修レコード自体の状態」と「申込受付状態」の両方を兼ねており、管理画面で登録した研修が会員側の申込一覧に出ない原因になり得た。

**修正内容**:
- `研修状態コード` を lifecycle として扱い、`DRAFT` / `PUBLISHED` / `CANCELLED` / `ARCHIVED` を正規値にした。
- 旧値 `OPEN` / `CLOSED` は既存データ互換のため `PUBLISHED` として読み替える。
- 申込可否は `申込開始日` / `申込締切日` / `開催日` / `定員` / `申込者数` から `applicationStatus` / `isApplicationOpen` として API 層で導出する。
- 会員画面、職員別研修画面、管理一覧、公開ポータル、申込APIの判定を同じ導出ロジックに統一した。
- 会員側の研修申込画面に「最新情報を取得」を追加し、管理者登録直後の stale data を手動再取得できるようにした。
- `scripts/build-gas.mjs` で `gas/member/index.html` も同期するよう修正した。member split への画面反映漏れを防止する。
