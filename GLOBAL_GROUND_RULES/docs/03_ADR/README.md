# ADR 運用ルール

ADR（Architecture Decision Record）は、重要な技術判断を残すための記録です。

## どんな時に書くか
- 新規ライブラリ導入
- 主要フレームワークの切替
- DB設計の大きな変更
- 認証 / 認可方式の変更
- キャッシュ、非同期、ジョブ基盤の採用
- 監視、セキュリティ、デプロイ方針の変更
- 将来の運用負債に影響する判断

## テンプレート
### タイトル
ADR-XXXX: [判断名]

### ステータス
- Proposed / Accepted / Superseded / Rejected

### 背景
- 何が問題だったか
- どんな制約があったか

### 判断
- 何を採用したか

### 代替案
- 検討した候補
- 採用しなかった理由

### 期待効果
- 何が改善するか

### トレードオフ
- 失うもの / 将来負債 / 運用コスト

### 影響範囲
- コード
- インフラ
- データ
- 運用
- セキュリティ

### ロールバック / 移行
- 戻し方
- 段階移行の方法

### 参照
- Issue / PR / Docs / 一次ソース
